# 8. コマンド基礎
ITに強くなるには、やはりコマンドラインを使えるようになるのが大事です。ということで、重要なコマンドを10個くらい紹介します。<br>
また、ターミナルにもプログラミング言語のように言語のようなものがある(厳密にはシェル)と言いましたが、今回はWindowsの標準であるPowerShellではなく`bash`, `zsh`での説明になります。

`bash`, `zsh`はコマンドが全部小文字のやつです。基本的に`PowerShell`を覚えなくても`bash`, `zsh`でのコマンド(UNIX記法)だけ覚えれば大丈夫です。

<a id="eight-one"></a>
## 8.1. コマンドのルール

個別のコマンドの説明になる前にターミナルでコマンドを入力するときの基本的な記法を説明します。<br>
コマンドの基本形は以下のようになっています。

```bash
コマンド名 [オプション] [引数]
```

例えば、`ls -l documents` の場合
- `ls` : コマンド名
- `-l` : オプション（詳細表示を指定）
- `documents` : 引数（対象のディレクトリ）

のようになっています。それぞれの要素の間には半角スペースを入れます。<br>
つまり、コマンドは **何をするか** を最初に書くということです。

(パイプ(|)というのが入ってくると「最初に何をするかを書く」の説明は怪しくなります。)

<a id="eight-one-one"></a>
### 8.1.1. オプションの種類
コマンドは独自のオプションを持っています。オプションはコマンドごとにあるため多すぎるので説明しませんが、`-`の1文字で使う短形式と`--`の2文字で使う長形式があることだけ知っておいてください。<br>
基本的にコマンドは短形式で使うことが多いですが、自作したCLIは未来の自分やAIがすぐに意味が分かるように長形式を使うことが多いです。

- 短形式オプション: 1文字で指定し、`-` を付けます。<br>
おまじないみたいな感じでとりあえずつけるみたいなことが多いです。

例えば`ls`は下のように`ls -a`のように使うことが多いです(設定で`ls`が常に`-a`付きで実行されるようにaliasというのを設定することも多いです。)
```bash
ls -a                # -a オプション, --allの略
ls -la                # -l と -a を組み合わせ, -lは--longの略
```

- 長形式オプション: 複数文字で指定し、`--` を付けます。

```bash
ls --long             # 短形式は-l
ls --all              # 長形式は-a
```

<a id="eight-one-two"></a>
### 8.1.2. 引数と標準出力

コマンドに対象を指定するのが「引数」です。コマンドの結果はターミナルに表示されたり、されなかったりします。

```bash
cat file.txt          # file.txt が引数
# ファイルの中身が標準出力としてターミナルに表示される
```

これは奥が深いです。

<a id="eight-two"></a>
## 8.2. 超基本コマンド

<a id="eight-two-one"></a>
### 8.2.1. 移動と確認 ― cd, ls
ターミナルで最も大事なことはdir管理です。自分がどこに行きたくて、そこに何があるのかを意識することは非常に大事です。

`cd`（change directory）はディレクトリを移動するコマンドです。相対dirでやった規則に従えば大丈夫です。また、引数がないときにhome dirに移動するという仕様があるのでそれをよく使います。

```bash
cd documents          # documentフォルダに移動
cd ../../             # 2階層上に移動
cd ~                  # home dirに移動
cd                    # 引数がないときはhome dirに移動
cd /                  # ルートディレクトリに移動
```

`ls`（list）はそのディレクトリの中身を表示するコマンドです。とはいえ、基本的には`ls`で`ls -a`として動作するように設定することが多いので、`ls`と`ls -la`しか使わない気がします。

```bash
ls                    # ファイル一覧を表示
ls -l                 # 詳細情報付きで表示
ls -la                # 隠しファイルも含めて詳細表示
```

また、慣れてくると使いませんが、今自分いるdirをにいるかを`pwd`（print working directory）で絶対パスで出すこともできます。

```bash
pwd
# 出力例: /Users/username/documents
```

<a id="eight-two-one-one"></a>
#### 8.2.1.1. 発展便利コマンド ― zoxide, fzf

<a id="eight-two-one-one-one"></a>
##### 8.2.1.1.1. z
デフォルトで使えるdirの移動コマンドは`cd`ですが、`cd`は直接文字を打つならcurrent dirより下のdirにしか行くことができず、そうでなければ`../`を並べないといけないというルールがあります。<br>
下に動く分には`cd`でもあんまり問題ないですが、上にいくつか戻るタイプの移動をするなら`cd` で`../`の数を考えながら打つのは非常に面倒です。

そんなときに外部からインストールする`zoxide`(z) と `fzf` を使うと、よく使うディレクトリへの移動が爆速になります。(僕の爆速環境構築セットには入っています。)

`zoxide`(コマンドとしては`z`) は移動履歴を学習して、
- 部分一致で
- どんな場所からでも

行きたいdirに素早くジャンプできます。1文字のコマンドなのも使いやすいです。

```bash
z documents    # 過去にアクセスしたdocumentsフォルダにジャンプ
z info         # 過去にinformation_processingにzで移動していれば、次からは少しの入力でも飛べる。
```

一方で、`docs/`や`src/`などの色々なプロジェクトで作るdirに`z`で飛ぶと「急に別プロジェクトの`src`に飛んだ」みたいなことになります。
なので「project dirには`z`」, 「そこからの移動は`cd`」みたいに運用を分けると良い気がします。

<a id="eight-two-one-one-two"></a>
##### 8.2.1.1.2. fzf

`fzf` はコマンドというよりは便利なショートカットキーの追加です。昔は`peco`ってやつが流行ってたらしいんですが、今はこれが主流です。<br>
便利で使ってて楽しいですが、最初は`z`だけでもいいと思います。(僕も最近使い出した)

特徴としては
- 深いdirの方のdir名の入力で
- 曖昧検索で(つまり誤字とかしても許される)
- 対話的に(最初はめっちゃ色んな候補が出てくるけど入力によって上の方に入力に近いdirやファイルが出てくる感じ)

ファイルやディレクトリを選択できます。

意味が分かりにくいと思いますし、実際に使ってみないと分かりにくいと思います。<br>
例えば、今`~`(home dir)にいて、`~/Documents/tuat/1.fyear_fsem/information_processing/`に移動したいってなったときに、`cd`だと<br>
「DocってTab押してTab補完, tuatって打って、1.って打ってTab補完, inって打ってTab補完」<br>
みたいに相当面倒なわけです。

`z`で移動することもできるんですが、`fzf`を使うなら
`Alt + C`を押してから「information_」とかって打てば1番上に「お前は`Documents/tuat/1.fyear_fsem/information_processing`に行きたいのか！？」みたいに出るわけですよね。これが便利です。

他には、この後紹介する`mv`コマンドで「`very/super/hyper/deep/dir/target_file.md`を移動したい」ってなったときに
1. `mv `って打つ
2. `Ctrl + T`を押して`target_file.md`って打つ
3. 「お前が操作したいのって`very/super/hyper/deep/dir/target_file.md`じゃね？」みたいに出てくる
4. それをEnterすると`mv very/super/hyper/deep/dir/target_file.md `の状態になる 

みたいな感じで入力がめっちゃ簡単になるわけです。

まあ他にも色々便利な使い方はありますし、最近は`ghq + fzf`というのが流行っていて他のツールとの連携でさらに便利になったりします。<br>

<a id="eight-two-two"></a>
### 8.2.2. ファイル, dirの操作 ― mv, cp, rm, mkdir

`cd`, `ls`とここで扱う`mv, cp, rm, mkdir`を覚えれば超基本コマンドとしては十分だと思います。`z`や`fzf`も活用すれば好き勝手にターミナルで遊べると思います。

`mv` (move) はファイルやフォルダを移動・リネームするやつです。<br>
`mv 動かしたいやつ 動かす目的地` です。

```bash
mv downloads/file.txt documents/various/          # documents/various dirに移動
mv 1.fyear_fsem/physical_chemistry 2.fyear_ssem/  # 間違えて1年前期の方に入れた物理化学のdirを1年後期の方に移動。

mv target_dir/* destination_dir/                  # target_dirの中身をdestination_dir/ に移動

mv old.txt new.txt   # ファイルをリネーム
mv old_dir/ new_dir/ # dirをリネーム(new_dirがまだ存在してなかったら)
```
**注意**
中身だけ動かしたいのに`mv target_dir/ destination_dir/`とすると(つまり、`target_dir/*`の`*`が無い場合)、例えば`target_file.txt`があったら<br>
`destination_dir/target_dir/target_file.txt`のように、`destination_dir`の下に`target_dir`ごと移動してしまます。<br>
なので中身だけ移動させたいなら`target_dir/*`のように書きます。

<a id="eight-two-two-one"></a>
#### 8.2.2.1. cp ― ファイル, dirのコピー
`cp` (copy) はファイルやフォルダをコピーする。`mv`は`mv target_dir destination_dir/`でオプションを付けなくてもdirを動かせるのに、`cp`はdirをコピーするなら`cp -r`としなくてはいけないのが特徴です。<br>
(`-r`は`--recursive`の略です。`-r`が必要な理由は僕は知りません。)<br>
その他の点はmvとほとんど同じです。

```bash
cp file.txt file_copy.txt    # ファイルをコピー
cp -r folder/ folder_copy/   # フォルダを再帰的にコピー（-r必須）
```

<a id="eight-two-two-one-one"></a>
##### 8.2.2.1.1. mkdir ― dirの作成

`mkdir` (make directory) はdirを作成します。深いdirを一気に作る時だけ`-p`が必要です。(`--parents`の略, 深くないときに`-p`をしてもエラーにはならない)<br>
しかし、面倒なので`mkdir`で`mkdir -p`として動作するようにaliasを設定していることも多いです。

```bash
mkdir documents              # documentsフォルダを作成
mkdir -p a/b/c               # 階層的にフォルダを作成
```

<a id="eight-two-two-one-two"></a>
##### 8.2.2.1.2. rm ― ファイルやdirの削除

`rm` (remove) はファイルやdirを削除します。基本コマンドですがこれまでに紹介したコマンドに比べて圧倒的に危険なコマンドです。<br>
エクスプローラーでの削除はゴミ箱に行きますが、`rm`コマンドの削除はゴミ箱を介さないので完全に世界から消し去ります。正直使わなくても良いかなって思います。<br>
ファイルの削除はyaziやnvimなどのツールでも安全にできるので、わざわざミスしたらとんでもないことになるコマンドは使わなくてもいいです。

そして、`rm`もdirの削除に関しては`-r`が必要です。`-r`が必要なおかげでhome dirごと完全に破壊するみたいなミスは軽減されています。<br>
また、僕は`rm`の動作を書き換えて危険なdirやファイルは消さないようにしていたりします。

とはいえ、 **`rm`を使うときは絶対に絶対パスや`~`を使ってで書かない** を徹底してほしいです。手が滑ってパソコン内のデータが全部消し飛ぶとか起きたらやばいので。

```bash
rm file.txt                  # ファイルを削除
rm -r folder/                # フォルダを再帰的に削除
rm -rf folder/               # 確認なしで強制削除（細心の注意で使う）
```

<a id="eight-two-three"></a>
### 8.2.3. 検索 ― find, grep

<a id="eight-two-three-one"></a>
#### 8.2.3.1. find ― ファイルやdirを探す
`find` はファイルを検索する機能です。実は僕はコマンドでの検索を使えません(使い方を覚えられない)。覚えるとしても、この後紹介する`fd`や`rg`を覚えると思います。

これまでの`cd`, `ls`, `mv`, `cp`などに比べて、普通に使う分にも覚えることが多いんですよね。

`find`コマンドは`find [どのdirの配下を] (--type [属性]) --name "名前"`
みたいに使います。

```bash
find . -name "*.txt"         # 現在のディレクトリ以下の.txtファイルを検索
find . -type f -name "*.md"  # ファイルのみ検索
find . -type d -name "test"  # ディレクトリのみ検索
```

<a id="eight-two-three-two"></a>
#### 8.2.3.2. grep ― ファイルの中の文字列を探す

`grep` は基本的にはファイルの内容から文字列を検索します。

`grep [オプション] "文字列" [対象のファイル]` のように使います。

```bash
grep "error" log.txt                  # log.txtから"error"を含む行を表示
grep -r "TODO" .                      # 現在のディレクトリ以下の全てのファイルに対して再帰検索
grep -i "warning" file.txt            # 大文字小文字を区別せずに検索
grep -n "import" script.py            # 行番号付きで表示
```

<a id="eight-two-three-three"></a>
#### 8.2.3.3. fd, rg

`find` と `grep`は古いコマンドで、最近は`fd` と `rg` (ripgrep)が流行ってきています。`fd`と`rg`の方が使い方が分かりやすいですし、速度も1000倍速かったりするらしいです。

`fd`は`find`の現代版です。`find`はcurrent dirより下を検索するときに`find .`が必要ですが、fdでは必要ありませんし、`--name`も不要です。

```bash
fd "\.txt$"                  # .txtファイルを検索
fd "config" --type f         # "config"という名前のファイルを検索
fd "test" --type d           # "test"という名前のディレクトリを検索
```

`rg` (ripgrep) は `grep` の現代版です。

```bash
rg "error"                   # 現在のディレクトリ以下から"error"を検索
rg "TODO" --type py          # Pythonファイルのみを検索
rg -i "warning"              # 大文字小文字を区別しない検索
rg "import" -n               # 行番号付きで表示
```

## 8.3. 応用

<a id="eight-three-one"></a>
### 8.3.1. ファイルの閲覧と編集 ― cat, nano

`cat`と`nano`はテキストエディタやIDEが無い状態で使うことが多いです。

`cat` (concatenate) はファイルの内容をターミナルに表示します。

```bash
cat file.txt                 # ファイルを表示
cat file1.txt file2.txt      # 複数ファイルを連結表示
```

`nano` はターミナル上のテキストエディタです。簡単なファイル編集ならこれで十分です。

```bash
nano file.txt                # ファイルを編集
nano new_file.txt            # 新しいファイルを作成して編集
```

`nano` の基本操作
- `Ctrl + O` → 保存（ファイル名を聞かれたらEnter）
- `Ctrl + X` → 終了
- `Ctrl + W` → 検索

<a id="eight-three-one-one"></a>
#### 8.3.1.1. eza, bat

より便利に使いたい人向けの補足です。

`eza` は `ls` の高機能な代替で、色分けやアイコンが付きます。`ls`のaliasに登録して`ls`が自動で`eza`として振舞うように設定することが多いので、手で`eza`を打つことは少ない気がします。

```bash
eza -l                       # 詳細表示（色付き）
eza -la                      # 隠しファイルも含める
eza -lT                      # ツリー表示
```

`bat` は `cat` の高機能版で、シンタックスハイライト(色)が付きます。
`bat`のオプションは`cat`のオプションと互換性がないのでaliasにできませんが、まあ入れておいて良いと思います。

```bash
bat file.txt                 # ファイルを表示（色付き）
bat --line-numbers file.txt  # 行番号付きで表示
```
