# プログラミング言語概論
ということで、環境構築は書ききれないのでまずはプログラミングの理論の方を書いていきます。<br>
まずは、簡単かつ典型的な書き方のpythonのファイルを見て、どのような要素でpythonのファイルが構成されているかを見てみましょう。あと、今後はですます調がめんどくさいのでである調に変わります。

## プログラミング言語の種類と違い
プログラミング言語は数多く存在する。例えば、Python、Rust、Kotlin、TypeScript、Go、あるいは関数型と呼ばれるHaskellなどがある。これらは大きく2つの軸で分類できる。**コンパイルするかしないか**と、**型を宣言するかしないか**である。

### コンパイル言語とインタープリタ言語
コンパイルとは、人間が書いたソースコードをCPUが直接実行できる機械語に変換する作業のことである。機械語というのは「コンピューターの共通言語」のようなものであり、機械語で書かれたファイルを **バイナリファイル** という。
電子的なコンピューターが発明されたときには機械語のみが存在していたため、全てのプログラミング言語は人類が意図的にバベルの塔で言語を散らばらせたという風に言えるかもしれない。

そして、**プログラミング言語から機械語への変換(コンパイル, バイナリ化)を事前に行う** 言語をコンパイル言語という。このコンパイルにより生成される機械語で書かれたファイルをバイナリ(+ファイル)と言う。<br>
一方で、**変換を行わず1行ずつ読みながら実行する** 言語を インタープリタ言語 という。

### 静的型付けと動的型付け
コンパイル ↔ インタープリタの対比と同じくらい重要な概念、というよりもコンパイルとインタープリタのそれぞれの言語設計の基盤となる概念が静的型付けと 動的型付けである。コンパイル言語は静的型付けを、インタープリタ言語は動的型付け言語であることがほとんどである。<br>

静的型付けというのは**コードの段階で「この変数の型(Pythonの解説で説明)」というのを宣言する(これを型宣言という)** ということであり、動的型付けは実行時にその変数の型が確定する。<br>
静的型付けによりコードの段階で型を決めていることで実行せずともコンパイルの段階でエラーが出る。

運用に近い観点で分類すると「コンパイル」「インタープリタ」であり、それを支える根源的な仕組みに着目すると「静的型付け言語」「動的型付け言語」に分類できる。

### コンパイル言語・静的型付け言語と、インタープリタ言語・動的型付け言語の利点と欠点
一般的に、コンパイル言語かつ静的型付け言語は、実行前にコンパイルが走るため、型の不一致や変数の未定義などの多くのエラーを実行前に検出できるため安全性が高い。また、コンピューターが読み取る最も根源的な(低レベルな)言語である機械語に変換済みであるため実行速度が速い。一方で、開発の段階で型宣言が必須であり、また変更したファイルを実行するためには毎回ビルドが必要な点でも開発速度は遅くなる傾向がある。

対して、インタープリタ言語かつ動的型付け言語は、型宣言が不要であり、コードを素早く書いて実行できるため、開発が簡単である。しかし、エラーは実行時に初めて発覚することが多く、実行速度も遅い。

具体例として、Rustはコンパイル言語かつ静的型付け言語の代表例であり、Pythonはインタープリタ言語かつ動的型付け言語である。<br>
Rustはメモリ安全性と実行速度が非常に優れていいるため、近年人気が高まっている。一方でPythonは開発が容易であるため初心者から企業まで広く技能レベル、そして広い領域で採用されている。

ただし、Pythonは型アノテーション(後述)を書き、それに型チェックを適用することで実行前に型のチェックができるため、静的型付けの強みを取り入れるような運用もできる。

### AI時代のプログラミング言語
近年、AIがコード生成を行うようになったことで、プログラミング言語の選択基準が変わりつつある。<br>
静的型付けでコンパイルが必要なRustは、他の言語にはない概念がありそれが学習難度に繋がっていた。しかし、生成したコードのエラーをビルド時に即座に検出できるためAIのコードに対しての認知負荷が軽減され、またビルドもAIが自動で行えるため開発速度も制約がなくなっている。このため、AIコーディングの普及によりでますますRustの人気が高まっている。

一方、Pythonは過去数十年のコード資産が膨大にあり、AIの学習データが豊富である。そのため、言語仕様自体にAIとの親和性は特にないが、それをコード資源で補って、現時点ではPythonもAIが書きやすい言語になっている。

# Python基礎(+理論)
## 簡単な例
``` python
# example.pyに書かれているとします。
def function(hop: int, step: int, jump: int) -> int:
  """3つの整数を足し合わせて返す関数"""
  fly_high = hop + step + jump
  return fly_high

if __name__ == (__main__)
  the_farthest_future = function(1, 2, 3)
  print(the_farthest_future)
 
#これを実行したときの標準出力
6
```

上のコードにおいて、`def function`から`return fly_high`までが関数の定義部分である。<br>
たとえば、パン工場で言えば、様々なパンが流れてきてそのパンに応じて袋詰めをして製品として出荷する。その袋詰めの仕組みに当たるのが関数の定義の部分である。

また、`function(hop, step, jump)`における`hop, step, jump`を引数(ひきすう)という。パン工場では流れてくるパンに対応した袋で包装するが、その時々の流れてくるパンが引数である。

`return fly_high`では関数の結果として`fly_high`という値が出力される。パン工場でいう袋詰めされた後の商品である。<br>
今回は整数が出力されるが、このドキュメントを通して **「値」は数値を意味しない。** また、今回のコードを含めてはじめは`return ○○`の形で値を返すことが多いが、返り値を返さないような関数もある。

`def function(hop: int, step: int, jump: int) -> int:`における`hop: int`と`-> int`の部分を型アノテーション(annotation)という。これは「`hop`という引数は`int`型である」「この関数の返り値は`int`型である」と引数の型と返り値の型を宣言している。<br>
型アノテーションがあってもPythoは動的型付け言語であるため型アノテーションは実行に影響しない(つまり、引数に型アノテーションと異なる型が与えられたりしても即座にエラーにはならない)が、コードを読む人間のコードの可読性を担保するほか、型チェッカーは型アノテーションをもとにエラーを検出することができる。

また、`"""3つの整数を足し合わせて返す関数"""`という部分をdocstring(ドキュメンテーション文字列)という。これは関数が何をするかを説明する文字列であり、`help(function)`で参照できたり、IDEの補完機能で表示されたりする。

`if __name__ == (__main__)`は実行のための呪文のようなもので、単体でこの`.py`ファイルを実行したときにのみ実行される内容が下に記述されている。これによりテストのしやすい設計を担保している。<br>
仮に`if __name__==(__main__)`が無い場合は`import`して用いた場合にも
```
the_farthest_future = function(1, 2, 3)
print(the_farthest_future)
```
の部分が実行されてしまう。

## クラスと型とクラス構造体
### 型という概念
プログラミングにおいて、値はすべて何らかの **型(type)** を持つ。型とは「その値がどのような種類のデータか」を示すラベルであり、かつ型に応じてその型が持つ「属性(attribution)」も規定される。

よく使う型の一覧を以下に示す。

| 型 | 意味・内容 |
|---|---|
| `str` | 文字列。`"hello"`, `'world'`など |
| `int` | 整数。`1`, `-5`, `100`など |
| `float` | 浮動小数点数。小数点が浮動する数。`3.14`, `-0.5`など |
| `bool` | 論理値型。booleanの略。`True`, `False`の2つの値のみが`bool`型である。 |
| `list` | 順序付きの可変長配列。`[1, 2, 3]`など |
| `tuple` | 順序付きの不変配列。`(1, 2, 3)`など |
| `dict` | キーと値のペア(連想配列)。`{"key": value}`など |
| `NoneType` | `None`のみを値として持つ型。値がないことを表す |

下のは少しマニアックな型である。

| `set` | (重複無しの)集合。`{1, 2, 3}`など |
| `frozenset` | `set`の不変版。変更不可 |
| `complex` | 複素数。`1+2j`など |
| `bytes` | バイト列。バイナリデータを扱う。`b"hello"`など |

### インスタンスと型
Pythonは動的型付け言語という話はしたが、それが具体的に何を意味するかというと **「インスタンス(値)が先に存在して、それに対して名前(変数名)をつけている」** ことである。
インスタンスというのはその型を持った実際の値(値$\neq$ 数)のことである。

例えば、以下のコードを考える。
```python
a = "hello"
```

初心者の人は「`a`は`"hello"`である」、少し知識がつくと「変数 `a` に文字列 `"hello"` を代入した」と考えるが、Pythonでは異なる。<br>
実際には

1. `str`型の`"hello"`というインスタンス(事物)がメモリ上に存在する。
2. `a = "hello"`で、その`"hello"`というインスタンスに`a`という名前をつけた。

というようになっている。

つまり、変数 `a` は値そのものではなく、値(`"hello"`)に対してつけた「名前」に過ぎない。このため、Pythonでは同じ変数に異なる型の値を再代入できる。
```python
a = "hello" # str型のインスタンス"hello"を参照
a = 10      # int型のインスタンス10を参照
a = [1, 2]  # list型のインスタンス[1, 2]を参照
```

また、Pythonは **本質的には「この変数の型はこれ」と決まることがなく、あくまで「このときのこの変数(+が参照しているインスタンスの型)はこの型」というその瞬間の型が決まっている** のみである。これが動的型付けの意味である。

対して、Rustなどの静的型付け言語では、変数の型が宣言時に決まり、実行中に変わらない：
```rust
let a: &str = "hello";  // aの型は&str（文字列型）と宣言
// a = 10;              // エラー：異なる型は代入できない
```

### クラスと構造体
クラスとは、「ある変数がどのような型で, どのように構成されているかを定義したもの」である。`str`もある変数が1つの`""`の形になっていると定義されるため、クラスである。

そして、pythonでは`class A:`でクラスを新たに定義できる。このように定義されたものをクラスと呼称することが多いが、`str`も`int`も`list`も`tuple`も新たに定義したクラスではないがクラスではある。<br>
よって、「クラス」という名称での混乱を避けるために`str`, `int`, `list`, `tuple`などの**ユーザーが新たに定義したわけではないクラス**に関しては**型**と呼ぶことにする。

そして、`class A:`で新たに定義したクラスを(このドキュメント内では)**クラス構造体**と呼ぶことにする。他の言語における構造体(structure)は複数のデータ型のみを持つクラスであり、Pythonの`class A:`で定義されたクラスはメソッドも持てるため、区別のためにクラス構造体と呼称する。

なお、オブジェクトはインスタンスを意味することもあれば(クラス)構造体を意味することもあり、意味が混同される可能性が高いためこのドキュメントでは用いない。

### 主要な型
#### `str`
stringの略もっとも身近な型の1つ。`"hello"`などの`""`や`''`の囲み文字を含めた文字列のことを表す。<br>
下のコードのように`+`演算子で文字列の長さを変えられていることから分かるように、`str`も可変長配列であり、イテラブルな型である。
```python
example = "hello"
example = example + " world"
print(example)  # "hello world"
```

#### `bool`
`boolean`の略で、日本語では論理値型という。`True`か`False`の2値しか取らない型である。<br>
`bool(x)`の挙動や各型の`__bool__`については [[#`bool(x)`と`__bool__`]] を参照。

#### `list`
順序付きの可変長配列。中の要素は任意の型を混在させることができる。<br>
`.append()`メソッドを用いて要素の数を増やすことができる。

```python
example = [1, 2, 3]
example.append(4)
print(example)  # [1, 2, 3, 4]
```
`[]`で囲んで作成する。代表的なイテラブルな型である。[[#イテラブル(Iterable)な型]]

#### `tuple`
順序付きの不変配列。`list`と異なり、作成後に要素を変更できない。

```python
example = (1, 2, 3)
example = ("a", "b", "c")
example = (1, "a", 3.14)
```
`()`で囲んで作成する。イテラブルな型である。[[#イテラブル(Iterable)な型]]

また、プログラミングとは直接関係はないが決まった個数の数値の組み合わせ（例えば座標`(x, y)`や`(a, b, c)`）をベクトルと呼ぶ。

#### `dict`
キーと値のペアを持つ連想配列。キーは文字列で、値は任意の型。
```python
example = {"a": 1, "b": 2, "c": 3}
example = {"name": "Taro", "age": 20}
```
`{}`で囲み、`key: value`の形で作成する。イテラブルな型である。[[#イテラブル(Iterable)な型]]

#### `set`
集合のような型で、重複を許さない(重複が除かれる)可変長配列である。<br>
下のコードのように`.add()`メソッドを用いて要素を加えることができる。
```python
example = {1, 2, 3}
example.add(4)
print(example)  # {1, 2, 3, 4}
```
`{}`では空の`set`ではなく空の`dict`を意味するため、空の`set`は`set()`関数で生成する。

### 配列について
配列とは、複数の値をまとめて格納するデータ構造のこと。配列を持つ型は全てイテラブルな型であり、`for x in A` で値を1つずつ取り出せる。[[#イテラブル(Iterable)な型]]

プログラミングで複数の値をまとめて扱う方法は大きく3つに分類できる。

可変長配列 — `list`、`set`、`str`。作成後に要素の追加・削除・変更が可能。実務では最も頻繁に使われる。

不変配列 — `tuple`。作成後に変更できない。関数の複数の戻り値や、変更されてはいけないデータを扱うときに使う。

連想配列 — `dict`。キーと値のペアで、キーを使って値にアクセス。設定ファイルやデータベースの行のような、複数の関連したデータをまとめるときに使う。

### イテラブル(Iterable)な型
イテラブルな型と言うのはイテラブル型という単一の型ではなく、値を1つずつ取り出せる型の総称である。<br>
`list, tuple, dict, set, str`などが挙げられる。また、`for x in A`で`A`に入れることができる型は全てイテラブルな型という判定方法がある。<br>
そして、イテラブルな型のインスタンスの各要素を取り出して並べたときの並びをイテラブルという。

今後はイテラブルな型全てを意味するときには`Iterable[T]`と表記し、特定の型のイテラブルであることを強調するときには`Iterable[int]`などで表す。

また、

```
example = [1, 2, 3]
```
は`Iterable[int]`の中の`list[int]`型である。

### `for`文の意味合い
`for`文は「繰り返し」と説明されることが多いが、厳密には**イテラブルな型の要素を1つずつ取り出して処理する** ことである。

```python
for x in example:
    print(x)
```

この場合、`in` の右側にはイテラブルな型（配列）が来る必要がある。<br>
`in` 演算子は「`example`の要素を1つずつ取り出す」という処理を行い、取り出された要素が `x` に代入されて、`for`ブロック内の処理が繰り返される。

つまり、`for x in A` が実行されるには、`A` がイテラブルな型である必要がある。`list, tuple, dict, set, str` などがイテラブルな型なので、これらに対して `for` 文を使うことができるのである。

#### Iterableな型へのアクセス
では、実際に

``` python
example = [1, 2, 3] #list
example = (1, 2, 3) #tuple
example = {1, 2, 3} #set
example = {"a": 1, "b":2, "c": 3}
example = "hello"
```
の5つの`example`に対して

``` python
for x in example: #exampleの中の要素を1つずつ取り出して表示する。
	print(x)
```
を実行した場合の出力を見てみる。
それぞれ以下のようになる。

#### 出力結果
```
#list
1
2
3

#tuple
1
2
3

#set
1
2
3

#dict
"a"
"b"
"c"

#str
"h"
"e"
"l"
"l"
"o"
```

#### `dict`型の値へのアクセス
上の出力結果から分かる通り、`dict`へのイテラブルなアクセスは値ではなくキーになっている。<br>
値にアクセスする場合は<br>
`example.values()`で値のイテラブル、`example.items()`で`("key", value)`の`tuple`のイテラブルが得られる。

#### `enumerate(Iterable[T]) -> Iterable[tuple(index, value)`
イテラブルなインスタンスに対して、それぞれの値に対してindexを付けたタプル形式のイテラブルを返す。
```python
example = ["a", "b", "c"]
for index, value in enumerate(example):
    print(index, value)
```
上のコードの出力結果は以下の通り。
```
0 a
1 b
2 c
```

### アノテーションと型チェック ― デフォルト値`None`の処理
アノテーション(処理には無関係だが変数や返り値の型を宣言しておくこと)は型チェックの対象となる。<br>
そして、型を厳密に扱う場合、 **`None`は`NoneType`型の(唯一の)値であり、デフォルト値として設定するときには注意が必要である。**

例えば、<br>
`def function(A:str = None) -> dict`<br>
(`str`型の`A`を受け取って`dict`を返す)
というのは厳密には間違っている。`None`は`NoneType`型であり、`str`型にはならないからであある。

厳密に正しく書くなら<br>
`def function(A:str | None = None) -> dict`が正しい。ここで、`|`は「または」を意味し、`str | None`のような表記をunionという。<br>
なお、Pythonの特殊な仕様として、`None`は型名は`NoneType`であるもののアノテーションでは型名を`None`と表記する。アノアテーションで型名を書かない唯一の例である。

###### 宣言型の言語の場合のデフォルト値`None`の扱い
Pythonでは`str`型の変数にデフォルト値として`None`を指定できるが、`str`と`None`は異なる型なので静的型付け言語では矛盾する。例えば、Rustでは`str`か`None`の両方の値を取りうる場合、`Option<&str>`というラッパー型で表現する。

```Rust
fn greet(name: Option<&str>) { }
```

このように、複数の型を取りうることを明示的に型で表現する必要がある。

##### 実は奥が深いprint関数
```
print(
	*values: Any,
    sep: str = " ",
    end: str = "\n",
    file: IO[str] | None = None,
    flush: bool = False,
    )
-> NoneType:
```

### 属性(attribute)という概念
`A.B`はAの中のBという属性を呼び出すもの。しかしながら、メソッド属性は属性の参照ではなく、Aの状態に基づいて機能を実行する呼び出しである。一方、モジュール内関数（例：`re.sub`）はモジュール関数として機能するため、`obj.method()`形式で呼び出すことはできない。

| 属性 | 意味 |
| --- | --- |
| データ属性 | obj.name | 値や変数(＝フィールド) |
| モジュール内関数 | re.sub | reモジュールの中の関数 |
| クラス属性 | MyClass.value | クラスレベルで定義される共通値。インスタンス属性により上書き可能 |
| メソッド属性 | obj.strip | 関数（そのオブジェクト専用の処理） |
また、`A.B()`は常に`B(A)`の同意味構文になるわけではない。例えば、
`int("25")`は成立するが`"25".int()`は成立しない。これは`int()`は型変換をするものであり、`str`は`int()`メソッドを持っていない。

```python
class A:
    value = 1          # クラス属性（デフォルト参照先）

    def __init__(self):
        self.value = 1 # インスタンス属性（各インスタンスに設定）

# 動作例
a = A()
print(a.value)  # 1（インスタンス属性が優先）
print(A.value)  # 1（クラス属性の初期値）

A.value = 2     # クラス属性を変更
print(A.value)  # 2（クラス属性が変更）

b = A()         # 新しいインスタンスを生成
print(b.value)  # 1（__init__でインスタンス属性が1に設定されるため）
print(A.value)  # 2（クラス属性は2のまま）
```

### データ属性が返り値を持てる
メソッド属性`A.B()`が返り値を持つことは理解しやすいが、実は`A.C`のようにデータ値属性も返り値を返すことがある。

例えば
``` python
df_year = df_day["date"].dt.year
```
という処理は、
①`df_day`という`DataFrame`が`"date"`というラベルの数値群を持っている
②その数値群は全て日付形式であり、それを全て年の情報だけに変換する
③そうしてできた新しい数値群に対して`"year"`というラベルをつけ、新しい数値群とともに`year`という名前の`Series`クラスのオブジェクトとして`DataFrame`に登録する
という流れになっている。

`=`という文の性質上、右辺はなんらかのオブジェクトになっているはずで、すなわち`.dt.year`というデータ値属性がなんらかのオブジェクトを返しているということを示している。実際に今回は`Series`クラスのオブジェクトを返している。

### `bool(x)`と`__bool__`
`bool(x: Any) -> bool`は内部的に`x.__bool__() -> bool`というメソッド属性を呼び出す。`__bool__`が定義されていない場合は`x.__len__() -> int`を呼び出し、`0`なら`False`、それ以外は`True`を返す。

よく使う型の`__bool__`の挙動は以下の通りである。

| 型 | `False`になる場合 | `True`になる場合 |
|---|---|---|
| `int` / `float` | `0`, `0.0` | それ以外の数値 |
| `str` | `""` (空文字列) | それ以外の文字列 |
| `list` / `tuple` / `set` / `dict` | 空(`[]`, `()`, `set()`, `{}`) | 要素が1つ以上ある |
| `NoneType` | 常に`False` | (なし) |

つまり`bool(x)`は`x == True`とは異なり、「空でないか・ゼロでないか」を判定するものである。
```python
bool(0)    # False
bool(1)    # True
bool("")   # False
bool("a")  # True
bool([])   # False
bool([0])  # True  ← 中身が0でもリスト自体は空でないのでTrue
bool(None) # False
```
