# 6. AI時代の基礎的な知識
<a id="6-1"></a>
## 6.1. AIと相性のいいファイルとは
AIの活用が大事な現代では、自分自身がプログラムを組むか組まないかに依らず「どのようにすればAIが活用しやすいか」を考えるようにすることが大事です。

活用しやすいというのは簡単に言うと「AIが使用するトークンが少なく、安定した出力を出せる」ということです。トークンというのは、まあ簡単に言うと計算に使った単語数みたいなものだと思ってください。これが少ないほど計算量が少ないということになります。

その中でも、最も基本的な話である「AI時代のファイルの在り方」について書こうと思います。<br>
ちなみに、AIが扱いやすい形式というのは`git`というシステムが扱いやすい形式でもあります。

また、AIを用いたプログラミングなどの、もっと踏み込んだ内容が知りたい方は[AIを使ったプログラミングの手法](#11)を参照してください。

<a id="6-1-1"></a>
### 6.1.1. 文章のファイルが最も扱いやすい
現時点において、AIが最も扱いのは文字による形式です。<br>
例えば、音声ファイルの「こんにちは」を「おはよう」に変えたり、画像ファイルに写った「こんにちは」を「おはよう」に書き換えたり、動画ファイルの3秒目から6秒目までに映った「こんにちは」を「おはよう」に変えるよりも、文章のファイルを書き換える方が遥かに簡単です。

なぜなら、音声ファイルや画像ファイル, 動画ファイルは内部ではバイナリファイル(0と1の並びで表現されたファイル)になっており、これを編集するには原理的にはその0と1の並びを編集することになるからです。バイナリファイルは基本的にかなり長い領域全体を見てようやく「ここがこうなってるな」って分かりますし、人間にとってのかなり些細な変更も01の文字列としてはかなり大きく変更されることになります。

そのため、自分の扱いたい一部分をちゃかちゃか編集するのはとても難しいです。(もちろん実際にはもっと賢くやってるんですが)

一方で、文章のファイルは文字、つまり文字に対応したunicodeの並びが登録されていますからそれを書き換えるだけで修正が完了します。自分の扱いたい情報がどこにあるかも分かりやすいですし、そこだけを書き換えることもできます。

<a id="6-1-1-1"></a>
#### 6.1.1.1. プレーンテキストのファイルが最も扱いやすい
上のsectionでは文章のファイルが扱いやすいという話をしましたが、文章のファイルならなんでもいいわけではありません。

例えば、みなさんが思い浮かべる文章のファイルと言えばワードファイルである`.docx`や、pdfファイル`.pdf`だと思いますが、これらは扱いずらいです。また、情報処理学の授業で用いられているPythonノートブックファイル`.ipynb`も扱いずらいです。

これらのファイルの特徴は「文字列以外のサブの情報が内部で入っている」という点です。<br>
`docs`ファイルや`pdf`ファイルは画像も入っていますし、「どこに」、つまり座標の情報がありますよね。他には、「ここは太字, ここは下線, ここは赤色」などの装飾の情報も入っています。

また、`.ipynb`ファイルは内部は`json`形式になっており、そこには「これが何番目のセルか」や「実行回数」などの情報が入っています。

これらの、人間が直接見ない構造全体を意味する情報を「メタデータ」と言うのですが、そういう情報が多ければ多いほどAIが扱いずらくなります。

これは「今のAIは情報全てを受け取ってしまい、それを捨てるということができない」ことで発生します。これをコンテキスト汚染といいます。<br>
`.docx`の場合は、内部が複数のファイルに分かれているので、さらに扱いずらいです。

一方で、`.txt`や`.md`はメタデータがなく、純粋に人間が見る文字がそのまま並んでいるだけなので、AIにとっての負荷が非常に少なくなっています。`.tex`ファイルも「これはpdfにするときにこういう風にしてね」という情報を与えますが、`.tex`ファイル自体はプレーンファイルですよね。だからこそ、レポートを書くときに普段から`.tex`で書いて`.pdf`にコンパイルするようにしていれば、AIに`.tex`ファイルを編集させて、AIと一緒にpdfのレポートを作ることもできるわけです。

### 6.1.2. AI時代のファイル名
これはファイル形式に比べてあんまり言うことが無いですが、「中身を見なくてもファイル名を読めば何が書いてあるかが分かる」というのが理想です。正直AIが無くてもそれを心掛けるべきなんですが、AI時代にはますます重要性が増しており、ちゃんとAI時代に生きるなら「必須」と言って良いレベルです。

先ほどから話していますが、今のAIは受け取った情報をそのまま全て受け取ってしまうので、人間がやっているような「これ見てみよ。あ、これ関係ないわ」としてそのファイルの内容をまともに受け取らない(「関係ない」ということだけ受け取る)ことができません。

なので、関係ないファイルを読むこと自体がコンテキスト汚染になりますし、自分の扱いたいことが載っているファイルを探すためにコマンドを叩いたりするのもトークンを消費するので、できるだけ最速で「お前が探してるファイルこれなんじゃね？」を出すことが大事です。

そのための工夫は無限にありますが、そのための最も初歩的なアプローチが「中身が分かるファイル名にする」ということです。<br>
正直AIのためならこれまでの人間用のファイル名よりも長い、50文字とかのファイル名の方が良いのかもしれませんが、僕はやっていません。

それでも中身が分かるファイル名をできる限り心掛けるのが大事ですし、また、関係ない内容を1つのファイルにまとめないなどの「責務の分離」が重要な理由もこれです。<br>
(「Bの処理どこに書いてあるの～、、、これここに書いてあんのかよファイル名は`A.py`だけどこの中にAだけじゃなくてBの処理も書いてあるじゃねーかよ」ってならないようにするためですね。)
