# 4. 初心者向けパソコン講座

<a id="4-1"></a>
## 4.1. ファイルやフォルダの扱い方
どうやらうちのボスの話によるとまだパソコンが苦手な人もわりかしいるようです。<br>
まずは、パソコンを扱う上で最も大事な、ファイル整理の話をします。

<a id="4-1-1"></a>
### 4.1.1. OneDriveを島流しにします
この補助資料の構成を何回考えても、やっぱりOneDriveを消し去らないと話が始まらない気がします。僕が書いた記事を見ながらやれば多分ミスはないはずなので、やっておくことを非常に強く推奨します。もちろん、「脳死で使っててもデータが残る」とはならなくなるので、データが消し飛ばないように気を付けないといけないですが、普通にGoogle Driveなどに大事なファイルは自分で保存したりして対策するのが良いと思います。

<a id="4-1-2"></a>
### 4.1.2. エクスプローラーを使いこなそう ― 覚えることは3つだけ
エクスプローラーというのはパソコンの下の方のバー(タスクバー)の左の方にある水色と黄色の、明らかにファイルを管理しそうなアイコンのやつです。

みんなに安心してほしいので言っておきますが、僕は普段からプログラミングをしますがエクスプローラーも使います。また、エクスプローラーの操作も普通にマウスでカチカチするのでエクスプローラーの操作は僕よりも速い人が沢山いると思います。

また、エクスプローラーの説明と書いたんですけど説明することはほとんどないです。普段使いの操作としては3つだけです。<br>
- 初期設定として **「拡張子の表示」, 「隠しファイルの表示」** をすること。
- 普段使いのための操作として **「困ったらPC→C:→ユーザー→[ユーザー名]」** と **「クイックアクセスへの追加」** と **「↑ボタン(1階層上に戻るボタン)」** <br>

のみです。つまり、普段使いのために覚えるのは3つだけです。

<a id="4-1-2-1"></a>
#### 4.1.2.1. 困ったときは(PC→)C:→ユーザー→[ユーザー名] ― 宇宙から星へ
まず、初心者の人が困ること第1位は「どこに何があるか分からない」だと思います。で、これを1番手っ取り早く解決する方法は決まった手順を何回も繰り返して覚えることです。

では、実際に手順を説明します。<br>
1. **まずエクスプローラーを開きましょう。**

開きましたかね？そうすると、画面の左の方に「ホーム」だの「ギャラリー」だの「デスクトップ」だのがあります。これはナビゲーションウィンドウというらしいですが、僕はサイドバーと呼びます。

2. **このサイドバーを下にスクロールすると「PC」というのがあります。**<br>
もしかしたらそのすぐ下に「よく分かんない文字列+(C:)」となってるやつ(今後C:と表記し、Cドライブと読みます)もあるかもしれません。<br>
**もし「PC」しかない場合はそれをクリックして、「よく分かんない文字列+(C:)」がある場合はそっちをクリックします。**<br>
**「PC」にアクセスした人はC:があるはずなのでクリックしてそこにアクセスしてください。**

今これで皆さんはC:というところにいるはずです。これは[4.2. ディレクトリの話をしよう](#5-2)で言うWindowsという宇宙の最高司令部ですね。<br>

3. **そこからは「ユーザー→ [ユーザー名]」と進んで下さい(ユーザー名は自分が一番よく使うやつです、分からないなら順番にアクセスしてどれか確かめましょう)。**<br>
下のフォルダが見えていればhome dirという場所(つまり星の最高司令部です、C:と同様に後で説明します)にきちんと進めています。
```text
/C:/Users/<ユーザー名>/ (= ~)
├─ Desktop
├─ Documents
├─ Downloads
├─ Music
├─ Pictures
├─ Videos
├─ AppData
│  ├─ Local
│  ├─ LocalLow
│  └─ Roaming
├─ OneDrive
├─ .local (最初の状態では存在しない)
│  └─ bin  
└─ (そのほかユーザーが作成したフォルダ)
```

4. **home dirに進めたらまたサイドバーの「PC」や「C:」をクリックして「ユーザー→ [ユーザー名]」と進んで、C:やhome dirに何があるかを覚えるまで繰り返してください。**

授業中に暇でかつdir構造を覚えていない方はずっとやってください。これをやった回数がその後のパソコンスキルに繋がってきます。

<a id="4-1-2-2"></a>
#### 4.1.2.2. 拡張子の表示, 隠しファイルの表示 ― 上のバーの方の「表示」で設定
まずは拡張子の表示や隠しファイルの表示をします。<br>
拡張子というのは`.pdf`, `.txt`, `.md`, `.py`, `.ipynb`, `.docx`などで、これは **「このファイルはこういう感じのファイルだよー」みたいなのを伝えるための識別子みたいなもの** です。

隠しファイルの表示はプログラミングに全く興味がない人はやらなくてもいいんですが、やって損はないのでやった方が良いとは思います。また、パソコンでマイクラがしたい方はやった方がいいです。

では実際の手順に移りましょう。

1. **とりあえずエクスプローラーを開いてください。**<br>
そうすると、「クイックアクセス」とか書かれてて「デスクトップ」だの「ダウンロード」だのよく見るフォルダが出てくると思います。

2. そして、 **そこから上の方に視線を動かすと「表示」とか書かれているのが見えると思います。**<br>
**そこをクリックすると1番下にさらに「表示」が出てきます。**<br>

3. **「表示」の中の「表示」をさらにクリックすることで「ファイル名拡張子」や「隠しファイルの表示」という項目が出てくるのでどちらもクリックしておきましょう。** チェックマークがつきます。

<details><summary>隠しファイルとは</summary>
  まず、「隠しファイルを表示」という設定ではありますが、隠しファイルのみならず隠しフォルダも表示されるようになります。<br>
一般的に隠しファイルは一般的には<code>.zshrc</code>, <code>.bashrc</code>, <code>.gitconfig</code>などの<code>.</code>始まりのファイルで、隠しフォルダも同様に<code>.local/</code>や<code>.ssh/</code>などの<code>.</code>始まりのフォルダなどといわれています 。
<strong>が、僕のパソコンだと<code>.</code>はじまりのファイルも隠しファイルを表示しない設定でも普通に表示されてるのでよく分かりません。</strong><br>
この後説明する「パス」でいう<code>/c/Recovery/</code>や　<code>/c/Users/[ユーザー名]/AppData</code>は隠しフォルダとしてWindowsにされており、隠しフォルダを表示にしないと表示されないのですが、それ以外はよく分かんないです。
</details>

#### 4.1.2.3. クイックアクセスにピン留め
今から説明するのは普段使いの操作です。<br>
「クイックアクセスへの追加」の話をします。

操作としてはサイドバーを使ってわちゃわちゃします。<br>

では手順を説明します。例えば「ドキュメント」dirに農工大のファイルを入れるdir(`~/Documents/tuat/`)を作って、この`~/Documents/tuat`をクイックアクセスに追加するという例で説明します。
```text
/C:/Users/<ユーザー名>/ (= ~)
├─ Desktop
├─ Documents
│  └─ tuat/
└─ (そのほかユーザーが作成したフォルダ)
```

1. **サイドバーからドキュメント」dirに飛んでください。**

2. **`tuat/`フォルダを探して"右クリック"して「クイックアクセスにピン留めする」をクリックしてください。**

3. **サイドバーを下にスクロールすると追加した`tuat/`dirが追加されているので次回から簡単にアクセスできます。**

同じように1年前期なら`1_fyear_fsem`(1年前期, 命名の仕方は[農工大のファイルを分けて整理しよう](#農工大のファイルを分けて整理しよう)に書いています)をクイックアクセスに追加して、後期になったらそれをクイックアクセスから外して`1_fyear_ssem`をクイックアクセスに追加するみたいにしておくと使いやすいです。

##### 4.1.2.3.1. OneDriveとの決別
OneDriveは消し去るべきですが、OneDriveの残滓(ざんし)が残っていないかの確認です。

なにを確認するかというと、デフォルトの状態のサイドバーにピン留めされている「デスクトップ」, 「ダウンロード」, 「ドキュメント」, 「ピクチャ」などなどが全部OneDriveの方のフォルダがピン留めされてるかの確認です。<br>
(つまり下の記事で言うドッペルゲンガーフォルダです)。

https://note.com/hirose_suguru/n/n70a347c77612

1. **サイドバーからピン留めされてる「ドキュメント」などにアクセスしてください。**

2. **画面の上の方に「OneDirve」って書いてないかを確認してください。**

3. **もし「OneDrive」の文字列があるならサイドバーからそのフォルダを右クリックして「クイックアクセスからピン留めを外す」で外して下さい。**

4. **そして、`PC`→`C:/Users/[ユーザー名]/`にアクセスしたところにある「ドキュメント」や「デスクトップ」をピン留めするようにしましょう。** こっちはドッペルゲンガーフォルダじゃない方です。

<a id="4-1-2-4"></a>
#### 4.1.2.4. 階層上に戻るボタン ― 上向きの矢印
では、普段使いの操作の2つめを説明します。
これができればとりあえずエクスプローラーの操作は大丈夫だと思います。

1. **まずエクスプローラーを開いてhome dirまで行ってみましょう**

2. **すると、上の方に「PC > C: > ユーザー > [ユーザー名]」と書かれてますよね。この部分をアドレスバーとかって言います。**<br>

3. **そこから目線を真横に向けると、上向きの矢印がありますよね。**

4. **これをクリックしてみましょう。画面はどうなりましたか？**

そうですね、「ユーザー」の画面に切り替わったと思います。さらにまた上向きの矢印をクリックしてみましょう。

C:に動いていると思います。つまり、この上向きの矢印(↑)は1つ上の階層に移ることを意味します。<br>

<details><summary>実は1階層上じゃない？</summary>
  実はこれは厳密には2階層上なんじゃないかという話があるんですよね。
  なんかそのファイルが存在するdir Aが1階層上で、dir Aが入っているdir B  は2階層上なんじゃないかみたいな。例えば、
  Pythonでは<code>Path</code>モジュールで同じdirじゃないdirのファイルを<code>import</code>しようとしたときとかに
  <code>
  str(Path(__file__).parent.parent)
  </code>
  みたいなコードが出てくるんですが、<code>parent</code>、つまり親dirが  2回出てきているのに これは僕たちの言う「1階層上」なんですよね。
  とはいえ、直感的に理解するためにこの資料では1階層上として扱います。
</details>

&#13;&#10;

**せっかくなので、C:→ユーザー→[ユーザー名]→ドキュメントまで行ったら上矢印で[ユーザー名]→ユーザー→C: に戻ってまた下に下がっていってを繰り返してみましょう。**

### 4.1.3. 農工大のファイルを分けて整理しよう
パソコンの良いところは「このファイルはここ」ってその人のやりたいように整理できることです。物理的な制約がないからです。<br>
なので、全てのファイルをてきとーに作っててきとーに管理しないようにしましょう。

たとえば、農工大のファイルを整理するとしましょう。プログラミングとかに全く興味がないならこういう感じにするのがいいと思います。
```
C:/Users/[ユーザー名]/Documents/農工大/
├── 1_1年前期/
│    ├── 細胞・遺伝学/
│    └── 情報処理学/
├── 2_1年後期/
│    ├── 生態・進化学/
│    └── 動物植物科学/
├── 2026_前期時間割.pdf
└── 2026_後期時間割.pdf
```

こんな感じで **4年生までを前期後期で分けて1～8までの数字を付けるのが良い感じな気がします。**<br>
なぜわざわざ番号を振るかと言うと、実は文字には番号が振られており(Unicode)、その順番だと **「前」** が **「後」** よりも番号が後ろにあるので、下のように **表示のときの順番が「後期」の方が上になって** めっちゃ気持ち悪いからです。

```
C:/Users/[ユーザー名]/Documents/農工大/
├── 1年後期/
│   ├── 生態・進化学/
│   └── 動物植物科学/
└── 1年前期/
    ├── 細胞・遺伝学/
    └── 情報処理学/

```
まあ気持ち悪いだけですし、逆に数字があるほうが嫌な方は無しでもいいと思います。並び替えのルールを作成順とかにしたらなんとかなるとも思いますし。

ちなみに、**もし数字をつけるなら絶対に1. 1のように間に空白を入れないで下さい。**<br>
**ファイル名やフォルダ名に空白を入れるのは絶対にやめた方がいいです。**

<a id="4-1-3-1"></a>
#### 4.1.3.1. もっとパソコン向けに整理するなら
プログラミングとかに興味がある人向けです。まあ興味なくてもパソコン的にはこっちの方が有難いのでやって損はないです。

**「パソコン向け」というのはどういうことかと言うと、「できる限り全て英語にする」ということです。**<br>
日本語はパソコンにおいてエラーの温床ですし、またEnterで確定してようやく入力したことになるので英語よりも検索とかが遅くなりがちです。

つまり、パソコン向けにもっと極端に作るなら全部英語にすればいいわけです。下みたいになります。
```
C:/Users/[ユーザー名]/Documents/tuat/
├── 1_fyear_fsem/                     # fyear: 1年, fsem: first semester, 前期
│    ├── cell_genestics/              # 細胞遺伝学
│    └── information_processing/      # 情報処理学
│
├── 2_fyear_ssem/                     # ssem: second semester, 後期
│    ├── ecology_evolution/           # 生態進化学
│    └── animals_plalnts/             # 動物植物科学
│
├── 2026_fsem_schedule.pdf
└── 2026_ssem_schedule.pdf
```
とはいえ、ここまでやるのは大変なので**農工大→tuat, 〇年〇期→〇_〇year_〇sem** くらいに留めて、科目名は日本語のままにしとくのがちょうどいいと思います。


<a id="4-1-3-2"></a>
#### 4.1.3.2. フォルダやとかファイルの命名で意識すること
ファイルの命名は実はトラップやしきたりが多いんですが、あんまり教えてもらうことがないと思います。<br>
なのでファイルやフォルダの命名するときに覚えておいた方がいいことを書いておきます。

- **空白の代わりにアンダースコア(`_`)を使え** : 空白はまじで使わない方がいいです。リボ払いと同じくらいやめた方がいいです。<br>
`2026_01_ファイルの命名方法.docx`とかそういう感じが推奨です。こうするだけで「お、こいつ分かっとるやん」ってパソコン詳しい人に好印象を持たれます。<br><br>
ちなみに、Windows自体がC:に`Program Files`という空白付きのdirを作っているせいでまじでよく不具合が起きます。頭がおかしいです。

- アンダースコア`_`とハイフン`-` : どっちがいいか分かりません。 **好みで良いと思います。** 僕はなんか気持ち的に`_`を使ってるんですが分かんないです。<br>
ちなみに、`.`で区切るのは拡張子と混ざるのでやめてください。

<details><summary>アンダースコアとハイフンは一応違いがある？</summary>
エクスプローラー上であるファイルをクリックして、0.5秒後くらいに撫でるようにクリックをすると、さらにその0.5秒後くらいに名前の変更のモードになります。(このラグがめっちゃ不愉快です)<br>
そのときに「`-`だと`-`と`-`の間の部分しか選択されなくて、`_`だと全部選択される」みたいな言説を見たんですが、手元のパソコンでやってみても「どっちも全部選択されてるやん」って感じで何も違いが分からないので、好みでいいと思います。<br>
また、URLはハイフンで区切るのが良いとされており、そのためGitHubのリポジトリもURLでもあるのでハイフンで区切りましょう。
</details
&#13;&#10;

&#13;&#10;
- **本当はできる限り英語が良い** : プログラミングとかをやりたい方は意識してほしいんですが、日本語は文字化けだとかエンコードというやつでのエラーがよく起きます。<br><br>
また、入力時に **「Ener」で確定しないと入力したことにならない** ので、打ちながらリアルタイムでファイルを探したりする機能(`zsh`のtab補完, `fzf`, `zoxide`など)のときに英語に比べて明らかに使いづらいです。<br>
そもそもターミナルというやつでコマンドを打つときは基本的に英語しか打たないのにそこで日本語入力に切り替えること自体が無駄な作業になるわけです。

## 4.2. ショートカットキーを覚えよう
ファイルの話が終わったので別の話に移ります。長かったですね。

スマートフォンやタブレットとパソコンの違いはショートカットキーがあることです。ショートカットキーは無限大にありますし、その大半は「これいつ使うねん」みたいなものだったりもします。<br>
[タイピングを簡単にしよう](#7-0-2)でも書いているんですが自分で設定して日本人向けに使いやすいように変えることもできます。

つまり、沼なんです。<br>
沼なんですが、今日は超重要なものだけをピックアップして伝えます。

- **Ctrl + c : コピー。** copyのc。ターミナルではコマンドの強制終了になるので、ターミナルではShift + Ctrl + cでコピーします。
- **Ctrl + x : 切り取り**
- **Ctrl + v : 貼り付け**
- **Shift + Ctrl + v : 書式なしで貼り付け。** Wordとかで太字だったり見出し扱いになってるテキストを、そういう装飾を無くした ただのテキストとして貼ることができます。
- **Ctrl + z : 1つ前に戻る。** 操作を間違えたときなどに戻すことができます。
- **Ctrl + y : 1つ後ろに進む。** Ctrl+zの逆。
- **Ctrl + s : 保存。** saveのs
- **Ctrl + f : 検索。** findのf
- **Ctrl + N : 新しいウィンドウもしくは新しいタブを開く。** newのn。Google ChromeやMicrosoft Edgeなどのブラウザでは新しいウィンドウを、VSCodeやWezTermでは新しいタブを開く(そういうやつはShift + Ctrl + Nで新しいウィンドウを開く)。
- **Ctrl + T : 新しいタブを開く。** tabのt。
- **Ctrl + B : 太字にする。** boldのb
- **Shift + Enter : 改行**
- **F7 : カタカナに変換**
- **Win(四角4つのキー) + I : 設定アプリを開く**
- **Win + x →キーを離してから u + u : パソコンをシャットダウン。** パソコンが固まったときも、マウスが動かなくてもキーボードは普通に反応することが多いんです。(マウスは画面内の座標を1秒間に何十回も計算するから割と処理が重いんです)。<br>
キーボードだけでシャットダウンしておく方法を覚えておくと便利です。
<br>

ちなみに、ショートカットキーが無限な理由は「アプリごとにショートカットキーが決まっているから」です。上に挙げたのはあくまで「一般的な挙動」であって、Ctrl + Kなどはアプリごとに動作がかなり違います。<br>
VSCodeというプログラミングによく用いられるソフトにはVSCode独自のショートカットキーの動作が決まっていますし、ターミナルなんてもっと一般的なソフトとは違ったショートカットキーをしています(Ctrl + cが分かりやすいですよね)。

ちなみに、CodexやClaude Code, Geminiなどはそれぞれ改行のキーがCtrl + EnterだったりShift + Enterだったりまちまちですが、全部Ctrl + jで改行できたりします。僕も最近知って感動しました。

## 4.3. 無駄なアプリを抜こう
無駄なアプリを抜きましょう！！<br>
WindowsパソコンはWinodwsの標準アプリに加えて、メーカーが突っ込んでいる謎のアプリが非常に多いです。docomoショップでスマホを買うとdocomoの意味わかんないアプリがめっちゃ入れられてるみたいなのと同じやつです。

そして、Windowsの標準アプリもいらないやつが多いです。実はWindowsにも「マップ」というアプリがありますが、Micorsoftのゴミみたいなマップアプリ使うならGoogleマップでいいですからね。

そして、最初からWindowsに入っているアプリ, メーカーに勝手に入れられたアプリは使ってるときに画面に出てくることも多いですし、また裏でめっちゃパソコンの計算能力を奪っていたりします。なるべく早く消し去っておきましょう。<br>
そもそも、パソコンに必要な必要なアプリはスマホよりも少ないですからね。

ではいきましょう！

<a id="4-3-1"></a>
### 4.3.1. アプリのアンインストール方法

一般的な手法と、外部ソフトを使ってもっと徹底的にアンインストールする手法があります。<br>
外部ソフトを使ってる人の理由はすごく正しいのですが、僕はそこまでしないので(そもそもあんまりアプリを入れないので)一般的な手法だけ使っています。

<a id="4-3-1-1"></a>
#### 4.3.1.1. 一般的な手法 ― Windowsの「アプリ」からアンインストール
1. **Win(Altとかの隣の四角4つのキー) + I(アイ)キーを押してください。** Windowsの設定が開きます。

2. **サイドバー(ナビゲーションウィンドウ)の「個人用設定」の下、「アカウント」の上に「アプリ」があるのでそれをクリックします。**

3. **「インストールされているアプリ」を開いてください。**

4. **アンインストールしたいアプリの右の方に「…」の三点リーダーがあるのでそれをクリックして「アンインストール」でアンインストールします。**

「アンインストール」が押せないやつはWindowsが入れてるやつで、消したらパソコン全体の動作に不具合が起きる可能性が高いやつです。<br>
ちなみに、アンインストールできないアプリでも3点リーダーのところで「詳細オプション」がある場合、そこから「バックグラウンド」の実行をどうするかの設定ができるので、それを用いましょう。

バックグラウンドというのは、そのアプリを開いていないときにも一部の機能を動かしておくみたいな感じです。例えば、メッセージアプリをバックグラウンドで動かしているからパソコンを立ち上げてからアプリを開いていなくてもメッセージが来たときに通知を出すことができます。

そこまで極端なことをしないとしても、アプリ側が「このフォルダだけ消そ」としていても実際には「ここにもお宅の関係ファイルありますやん」ってなることがあり、それは残り続けるわけです。

## 4.4. パソコンの今の処理状況を見よう
多くの人はメモリが16GBのパソコンを使っている気がします(生協パソコンは16GB)です。ノートパソコンにおいてもっとも律速的、つまりパソコンが「足りないよー」ってなるのはメモリの部分です。<br>

アプリを沢山開いているとメモリを使いきって処理が重くなるので、どのアプリがどれくらいメモリを食っているかを見れるようにしましょう。

1. **Ctrl + Shift + Esc** を押すと「タスクマネージャー」というのが出てきます。

2. **タスクバー(画面の下の方のバー)では水色と青のアイコンです。タスクバーのそのアイコンを右クリックして「タスクバーにピン留めする」としてピン留めしておきましょう。**

3. **メモリの部分を見ればどのアプリが重いかぱっと分かります。**<br>
多くの場合はGoogle ChromeやMicrosoft Edgeが1番重いことが多いです。これらのブラウザアプリはめっちゃ重いんですよね。

## 4.5. ブラウザを決めよう ― スマホと統一すればなんでもいい

<a id="4-5-1"></a>
### 4.5.1. ブラウザと検索エンジン
まずは我らがゆるコンピューター科学ラジオの動画を貼っておきます。

> 機械オンチが使うべきブラウザを発表します。<br>
https://youtu.be/FkDsEaWSlgk?si=JDG0yM6YfG0yQ3V4

僕の方でもブラウザとは何かについて話します。<br>
ブラウザの具体例はGoogle Chrome, Microsoft Edgeなどです。

ちなみに、googleやbing, yahooなどは「検索エンジン」というやつで、「この検索ワードを入れたときにどのサイトをどの順番で表示するか」を制御してるやつです。ブラウザとは別物です。

ブラウザではpdfを開いたり、他には自分で作っているアプリを動かして表示させることもできますよね。つまり、ネットで検索したりというのはブラウザの機能の一部でしかないわけです。<br>
今後はブラウザと検索エンジンを混同しないようにしましょう。

<a id="4-5-2"></a>
### 4.5.2. ブラウザ選び ― スマホと統一すればなんでもいい
では具体的にどのブラウザが良いかの話をします。
ブラウザは最も有名なGoogle Chrome, Microsoft Edgeがあり、次に有名なFiereFox, 僕が使っているVivaldi, 他にはBrave, Arc, OpenAIが作ったAIブラウザとか言われているAtlasなどがあります。スマホで言えば、iPhoneの標準ブラウザはSafariです。

まず、ゆるコンピューター科学ラジオでも「普通の人はダサいのを使えばいい」と言っていましたね。僕もまあ概ね同じような感じです。

かつ僕の場合は **「スマホと同じブラウザを使うならなんでもいい」** というのを伝えておこうと思います。

つまり、iPhoneの標準のブラウザであるSafariをWindowsで使う奇人はいないので、「**Windowsを使う時点でパソコンとスマホのブラウザを統一できないからiPhoneでSafariを使うのやめろよ** 」ということですね。

なぜ統一しないといけないかと言うと、**ブラウザが異なると 「パスワードの共有」 , 「履歴の共有」ができないからです。** 1passwordという課金して使うパスワード管理サービスを使うなら関係ないですが僕もそれは使ってないので、そうなると絶対に統一すべきです。

で、それ以上のブラウザの話はあんまり気にしなくて良いと思います。<br>
Chromeのメモリ使用量が多いというのは有名ですが、他のブラウザもそこまで大差ないんですよね。(理由は[ブラウザがブラウザとして動くために](#14-1)に書いています)

僕はVivaldiが好きなんですけど、VivaldiだってChromeと体感で明らかに差を感じるほど軽いわけではないし単に自分で好きなようにカスタマイズできるのが好きってだけです。<br>
