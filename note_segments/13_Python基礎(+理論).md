## 13.1. 簡単な例
``` python
# example.pyに書かれているとします。
def function(hop: int, step: int, jump: int) -> int:
  """3つの整数を足し合わせて返す関数"""
  fly_high = hop + step + jump
  return fly_high

if __name__ == (__main__)
  the_farthest_future = function(1, 2, 3)
  print(the_farthest_future)
 
#これを実行したときの標準出力
6
```

- 上のコードにおいて、`def function`から`return fly_high`までが関数の定義部分である。<br>
たとえば、パン工場で言えば、様々なパンが流れてきてそのパンに応じて袋詰めをして製品として出荷する。その袋詰めの仕組みに当たるのが関数の定義の部分である。

- また、`function(hop, step, jump)`における`hop, step, jump`を引数(ひきすう)という。パン工場では流れてくるパンに対応した袋で包装するが、その時々の流れてくるパンが引数である。

- `return fly_high`では関数の結果として`fly_high`という値が出力される。パン工場でいう袋詰めされた後の商品である。<br>
今回は整数が出力されるが、このドキュメントを通して **「値」は数値を意味しない。** また、今回のコードを含めてはじめは`return ○○`の形で値を返すことが多いが、返り値を返さないような関数もある。

- 引数である`hop`, `step`, `jump`と関数内で定義された`fly_high`をまとめて「変数(variable)」という。特に、関数の部分で定義された変数であるため「ローカル変数」という。

`def function(hop: int, step: int, jump: int) -> int:`における`hop: int`と(`step`, `jump`も同様)`-> int`の部分を型アノテーション(annotation)という。これは「`hop`という引数は`int`型である」「この関数の返り値は`int`型である」と引数の型と返り値の型を宣言している。

型アノテーションがあってもPythonは動的型付け言語であるため型アノテーションは実行に影響しない(つまり、引数に型アノテーションと異なる型が与えられたりしても即座にエラーにはならない)が、コードを読む人間のコードの可読性を担保するほか、型チェッカーは型アノテーションをもとにエラーを検出することができる。

また、`"""3つの整数を足し合わせて返す関数"""`という部分をdocstring(ドキュメンテーション文字列)という。これは関数が何をするかを説明する文字列であり、`help(function)`で参照できたり、IDEの補完機能で表示されたりする。

`if __name__ == (__main__)`は実行のための呪文のようなもので、単体でこの`.py`ファイルを実行したときにのみ実行される内容が下に記述されている。これによりテストのしやすい設計を担保している。<br>
仮に`if __name__==(__main__)`が無い場合はこの`.py`を`import`して用いた場合にも
```
the_farthest_future = function(1, 2, 3)
print(the_farthest_future)
```
の部分が実行されてしまう。

## 13.2. クラスと型とクラス構造体

<a id="13-2-1"></a>
### 13.2.1. 型という概念
プログラミングにおいて、値はすべて何らかの型(type)を持つ。 **型とは「その値がどのような種類のデータか」を示すラベル** であり、かつ型に応じてその型が持つ「属性(attribution)」も規定される([属性(attribute)という概念](#eleven-three-three))。

よく使う型の一覧を以下に示す。

| 型 | 意味・内容 |
|---|---|
| `str` | 文字列。stringの略。`"hello"`, `'world'`など |
| `int` | 整数。integerの略。`1`, `-5`, `100`など |
| `float` | 浮動小数点数。floating-point numberの略。小数点が浮動する数。`3.14`, `-0.5`など |
| `bool` | 論理値型。booleanの略。`True`, `False`の2つの値のみが`bool`型である。 |
| `list` | 順序付きの可変長配列。`[1, 2, 3]`など |
| `tuple` | 順序付きの不変配列。`(1, 2, 3)`など |
| `dict` | キーと値のペア(連想配列)。`{"key": value}`など |
| `NoneType` | `None`のみを値として持つ型。値がないことを表す。 |

下のは少しマニアックな型である。

| `set` | (重複無しの)集合。`{1, 2, 3}`など |
| `frozenset` | `set`の不変版。変更不可 |
| `complex` | 複素数。`1+2j`など |
| `bytes` | バイト列。バイナリデータを扱う。`b"hello"`など |

<a id="13-2-2"></a>
### 13.2.2. インスタンスと型
Pythonは動的型付け言語という話はしたが、それが具体的に何を意味するかというと **「インスタンス(値)が先に存在して、それに対して名前(変数名)をつけている」** ことである。
インスタンスというのはその型を持った実際の値(値$\neq$ 数)のことである。

例えば、以下のコードを考える。
```python
a = "hello"
```

初心者の人は「`a`は`"hello"`である」、少し知識がつくと「変数 `a` に文字列 `"hello"` を代入した」と考えるが、Pythonでは異なる。<br>
実際には

1. `str`型の`"hello"`というインスタンス(事物)がメモリ上に存在する。
2. `a = "hello"`で、その`"hello"`というインスタンスに`a`という名前をつけた。

というようになっている。

つまり、変数 `a` は値そのものではなく、値(`"hello"`)に対してつけた「名前」に過ぎない。<br>
比喩的に捉えるなら、式の右辺はインスタンスを世界に顕現させ、左辺はそれに名前をつけていると言える。

このため、Pythonでは同じ変数に異なる型の値を再代入できる。
```python
a = "hello" # str型のインスタンス"hello"を参照
a = 10      # int型のインスタンス10を参照
a = [1, 2]  # list型のインスタンス[1, 2]を参照
```

また、Pythonは **本質的には「この変数の型はこれ」と決まることがなく、あくまで「このときのこの変数(+が参照しているインスタンスの型)はこの型」というその瞬間の型が決まっている** のみである。これが動的型付けの意味である。

対して、Rustなどの静的型付け言語では、変数の型が宣言時に決まり、実行中に変わらない：
```rust
let a: &str = "hello";  // aの型は&str（文字列型）と宣言
// a = 10;              // エラー：異なる型は代入できない
```

### 13.2.3. クラスと構造体
クラスとは、「ある変数がどのような型で, どのように構成されているかを定義したもの」である。`str`もある変数が1つの`""`の形になっていると定義されるため、クラスである。

そして、Pythonでは`class A:`でクラスを新たに定義できる。このように定義されたものをクラスと呼称することが多いが、`str`も`int`も`list`も`tuple`も新たに定義したクラスではないがクラスではある。<br>
よって、「クラス」という名称での混乱を避けるために`str`, `int`, `list`, `tuple`などの**ユーザーが新たに定義したわけではないクラス**に関しては**型**と呼ぶことにする。

そして、`class A:`で新たに定義したクラスを(このドキュメント内では)**クラス構造体**と呼ぶことにする。他の言語における構造体(structure)は複数のデータ属性のみを持つクラスであり、Pythonの`class A:`で定義されたクラスはメソッドも持てるため、区別のためにクラス構造体と呼称する。<br>

詳しくはデータ属性, メソッド属性の説明 → [属性(attribute)という概念](#eleven-three-three)を見てもらいたいが、一旦クラス構造体と構造体のイメージがつくように具体例で考えてみる。<br>
例えば下の`Student`は、データ属性(`id`, `name`, `year_level`)とメソッド属性(`introduce`)を持つため、`Student`クラスはクラス構造体である。
```python
class Student:
    def __init__(self, name, score): #一旦理解せず無視してOK
        # self.〇〇でStudentクラスの持つデータ属性を定義
        self.id = id                  # データ属性
        self.name = name              # データ属性
        self.year_level = year_level  # データ属性

    def introduce(self):      # メソッド属性
        return f"My name is {self.name}."
```
もし`id`や`name`のようなデータ属性だけを保持し、メソッドを持たないなら、他言語の意味での構造体に近い。

例えば、Rustにはクラス構造体はなく単なる構造体しかない。次のように`struct`でデータ属性のみを定義する。<br>
メソッド属性も定義したい場合は構造体とは別で`impl クラス名`で別に書く。
```rust
struct Student {
    id: u32,  // 0 ～ 4,294,967,295の非負整数
    name: String,
    year_level: u8, // 0 ～ 255の非負整数
}

impl Student {
    fn introduce(&self) -> String { // メソッド属性
        format!("My name is {}.", self.name) // これがreturnにもなる
    }
}
```

なお、「オブジェクト」という用語があるが、オブジェクトはインスタンスを意味することもあれば(クラス)構造体を意味することもあり、意味が混同される可能性が高いためこのドキュメントでは用いない。

### 13.2.4. 主要な型

<a id="13-2-4-1"></a>
#### 13.2.4.1. `str`
stringの略もっとも身近な型の1つ。`"hello"`などの`""`や`''`の囲み文字を含めた文字列のことを表す。<br>
下のコードのように`+`演算子で文字列の長さを変えられていることから分かるように、`str`も可変長配列であり、イテラブルな型である。([配列とイテラブル](#13-2-5))

```python
example = "hello"
example = example + " world"
print(example)  # "hello world"
```

<a id="13-2-4-2"></a>
#### 13.2.4.2. `bool`
`boolean`の略で、日本語では論理値型という。`True`か`False`の2値しか取らない型である。<br>
各型の`__bool__`については [インスタンスが`bool`(真偽値)を持つ ― `__bool__()`メソッド](#eleven-three-five)を参照。

<a id="13-2-4-3"></a>
#### 13.2.4.3. `list`
順序付きの可変長配列。中の要素は任意の型を混在させることができる。<br>
`.append()`メソッドを用いて要素の数を増やすことができる。([属性(attribute)という概念](#eleven-three-three))

```python
example = [1, 2, 3]
example.append(4)
print(example)  # [1, 2, 3, 4]
```
`[]`で囲んで作成する。代表的なイテラブルな型である。([配列とイテラブル](#eleven-two-five))

<a id="13-2-4-4"></a>
#### 13.2.4.4. `tuple`
順序付きの不変配列。`list`と異なり、作成後に要素を変更できない。

```python
example = (1, 2, 3)
example = ("a", "b", "c")
example = (1, "a", 3.14)
```
`()`で囲んで作成する。イテラブルな型である。([配列とイテラブル](#eleven-two-five))

また、プログラミングとは直接関係はないが決まった個数の数値の組み合わせ（例えば座標`(x, y)`や`(a, b, c)`）をベクトルと呼ぶ。

<a id="13-2-4-5"></a>
#### 13.2.4.5. `dict`
キーと値のペアを持つ連想配列。キーは文字列で、値は任意の型。
```python
example = {"a": 1, "b": 2, "c": 3}
example = {"name": "Taro", "age": 20}
```
`{}`で囲み、`key: value`の形で作成する。イテラブルな型である。[[#イテラブル(Iterable)な型]]

<a id="13-2-4-6"></a>
#### 13.2.4.6. `set`
集合のような型で、重複を許さない(重複が除かれる)可変長配列である。<br>
下のコードのように`.add()`メソッドを用いて要素を加えることができる。([属性(attribute)という概念](#eleven-three-three))
```python
example = {1, 2, 3}
example.add(4)
print(example)  # {1, 2, 3, 4}
```
`{}`では空の`set`ではなく空の`dict`を意味するため、空の`set`は`set()`関数で生成する。

<a id="13-2-5"></a>
### 13.2.5. 配列とイテラブル
配列とは、複数の値をまとめて格納するデータ構造のこと。配列を持つ型は全てイテラブルな型であり、`for x in A` で値を1つずつ取り出せる。[[#イテラブル(Iterable)な型]]

プログラミングで複数の値をまとめて扱う方法は大きく3つに分類できる。

可変長配列 — `list`、`set`、`str`。作成後に要素の追加・削除・変更が可能。実務では最も頻繁に使われる。

不変配列 — `tuple`。作成後に変更できない。関数の複数の戻り値や、変更されてはいけないデータを扱うときに使う。

連想配列 — `dict`。キーと値のペアで、キーを使って値にアクセス。設定ファイルやデータベースの行のような、複数の関連したデータをまとめるときに使う。

<a id="13-2-6"></a>
### 13.2.6. イテラブル(Iterable)な型
イテラブルな型と言うのはイテラブル型という単一の型ではなく、値を1つずつ取り出せる型の総称である。<br>
`list, tuple, dict, set, str`などが挙げられる。また、`for x in A`で`A`に入れることができる型は全てイテラブルな型である。<br>
そして、イテラブルな型のインスタンスの各要素を取り出して並べたときの並びをイテラブルという。

今後はイテラブルな型全てを意味するときには`Iterable[T]`と表記し、特定の型のイテラブルであることを強調するときには`Iterable[int]`などで表す。

また、
```
example = [1, 2, 3]
```
は`Iterable[int]`の中の`list[int]`型である。

<a id="13-2-7"></a>
### 13.2.7. `for`文の意味合い
`for`文は「繰り返し」と説明されることが多いが、厳密には**イテラブルな型の要素を1つずつ取り出して処理する** ことである。

```python
for x in example:
    print(x)
```

この場合、`in` の右側にはイテラブルな型（配列）が来る必要がある。<br>
`in` 演算子は「`example`の要素を1つずつ取り出す」という処理を行い、取り出された要素が `x` に代入されて、`for`ブロック内の処理が繰り返される。

つまり、`for x in A` が実行されるには、`A` がイテラブルな型である必要がある。`list, tuple, dict, set, str` などがイテラブルな型なので、これらに対して `for` 文を使うことができるのである。

<a id="13-2-7-1"></a>
#### 13.2.7.1. Iterableな型へのアクセス
では、実際に

``` python
example = [1, 2, 3] #list
example = (1, 2, 3) #tuple
example = {1, 2, 3} #set
example = {"a": 1, "b":2, "c": 3} #dict
example = "hello" #str
```
の5つの`example`に対して

``` python
for x in example: #exampleの中の要素を1つずつ取り出して表示する。
	print(x)
```
を実行した場合の出力を見てみる。
それぞれ以下のようになる。

値にアクセスする場合は<br>

<a id="13-2-7-4"></a>
#### 13.2.7.4. `enumerate(Iterable[T]) -> Iterable[tuple(index, value)`
イテラブルなインスタンスに対して、それぞれの値に対してindexを付けたタプル形式のイテラブルを返す。
```python
example = ["a", "b", "c"]
for index, value in enumerate(example):
    print(index, value)
```
上のコードの出力結果は以下の通り。
```
0 a
1 b
2 c
```

<a id="13-2-8"></a>
### 13.2.8. アノテーションと型チェック ― デフォルト値`None`の処理
アノテーション(処理には無関係だが変数や返り値の型を宣言しておくこと)は型チェックの対象となる。<br>
そして、型を厳密に扱う場合、 **`None`は`NoneType`型の(唯一の)値であり、デフォルト値として設定するときには注意が必要である。**

例えば、<br>
`def function(A:str = None) -> dict`<br>
(`str`型の`A`を受け取って`dict`を返す)
というのは厳密には間違っている。`None`は`NoneType`型であり、`str`型にはならないからであある。

厳密に正しく書くなら<br>
`def function(A:str | None = None) -> dict`が正しい。ここで、`|`は「または」を意味し、`str | None`のような表記をunionという。<br>
なお、Pythonの特殊な仕様として、`None`は型名は`NoneType`であるもののアノテーションでは型名を`None`と表記する。アノアテーションで型名を書かない唯一の例である。

<a id="13-2-8-1"></a>
#### 13.2.8.1. 宣言型の言語の場合のデフォルト値`None`の扱い
Pythonでは`str`型の変数にデフォルト値として`None`を指定できるが、`str`と`None`は異なる型なので静的型付け言語では矛盾する。例えば、Rustでは`str`か`None`の両方の値を取りうる場合、`Option<&str>`というラッパー型で表現する。

```Rust
fn greet(name: Option<&str>) { }
```

このように、複数の型を取りうることを明示的に型で表現する必要がある。

<a id="13-2-9"></a>
### 13.2.9. コラム ― 実は奥が深いprint関数
```
print(
	*values: Any,
    sep: str = " ",
    end: str = "\n",
    file: IO[str] | None = None,
    flush: bool = False,
    )
-> NoneType:
```

`print()`関数が一般的すぎるあまり意識されることがないが、`print`関数は`*values: Any` とあるように **どんな型の値でも、そしてどんな長さの値でも可変長引数として受け取ることができる。** 整数、文字列、リスト、辞書、自己定義クラスのインスタンスなど、Pythonのあらゆるインスタンスをそのまま渡せる。<br>
一方、返り値は `NoneType`（つまり `None`）であり、print関数は値を返さない。その代わりに、与えられた値を整形して `file` パラメータ（デフォルトは標準出力 `sys.stdout`）に直接出力する。

可変長のイテラブルを受け取れる関数はいくつかあり、例えば`max()`(与えられた配列の最大値を返す), `min()`(与えられた配列の最小値を返す), `zip()`(2つのイテラブルを対応させる), `str.join()`(文字列を結合する)などは可変長のイテラブルを受け取ることができるが、あらゆる型を受け取ることができるわけではない。**あらゆる型の、そして可変長の値を受け取ることができる** のが`print()`関数の最大の特徴だと言える。

## 13.3. オブジェクト指向と属性

<a id="13-3-1"></a>
### 13.3.1. オブジェクト指向とは
オブジェクト指向(Object-Oriented Programming, OOP)とは、プログラムを構成する基本単位を「オブジェクト(データとそれに対する処理をまとめたもの)」として捉えるプログラミングパラダイムである。

Pythonはオブジェクト指向の言語である。

従来の手続き型プログラミングでは、データと処理が分離していた。例えば、`int`や`str`のインスタンスがあり、それらに対して関数を適用するという形式である。<br>
一方、オブジェクト指向では、「ある概念が本来持っているべき変数(データ属性)と処理(メソッド)を1つの単位にまとめる」ことで、現実世界の構造をコードに反映させる。プログラミング的には、**「あるクラス(型)がデータやそのクラスに適した処理を持つ」** という点で特徴的である。

例えば、銀行口座を考えると
- 手続き型：`balance`という値と`withdraw(balance, amount)`という関数を分けて管理
- オブジェクト指向：`BankAccount`というクラス(上位の概念)に残高というデータと、引き出し処理というメソッドを両方持たせる

このデータと処理をまとめたものを属性(attribute)と呼び、属性がオブジェクト指向の中心概念となる。データ属性（フィールド）とメソッド属性（関数）の2つからなる。

<a id="13-3-2"></a>
### 13.3.2. オブジェクト指向の3大要素
オブジェクト指向には以下の3つの重要な要素がある。

1. カプセル化(Encapsulation)
データと処理を1つの単位にまとめ、外部からの直接アクセスを制限する。これにより、オブジェクト内部の状態を保護し、一貫性を保つ。

2. 継承(Inheritance)
既存のクラスを基にして、新しいクラスを作成する。共通部分を親クラスにまとめ、差分を子クラスに記述することで、コードの再利用と拡張を実現する。

3. ポリモーフィズム(Polymorphism)
同じメソッド名で異なる動作を実現すること。例えば、複数の型が同じ名前のメソッドを持つとき、呼び出し側はその型を気にせず同じメソッド名で処理できる。

<a id="13-3-3"></a>
### 13.3.3. 属性(attribute)という概念
`A.B`はAがBという属性を呼び出すことを意味する。属性はクラス属性とインスタンス属性に分かれる。<br>

<a id="13-3-3-1"></a>
#### 13.3.3.1. クラス属性とインスタンス属性

<a id="13-3-3-1-1"></a>
##### 13.3.3.1.1. クラス属性
クラス属性はクラス、つまり設計図自体が値を決めている属性である。つまり、「インスタンスごとにここは変わるけど、これは共通だな」という部分がクラス属性である。クラス属性にはクラスデータ属性とメソッド属性がある。<br>
(クラスデータ属性は筆者の造語であり、メソッド属性と区別して作った。クラスデータ属性のことをクラス属性と呼ぶ人もいる。)

| 属性 | 書き方, 例 | 意味 |
| --- | --- | --- |
| クラスデータ属性 | MyClass.name | クラス全体で共通の値 |
| メソッド属性 | MyClass.strip() | クラス共通の関数 |

メソッド属性はインスタンスごとに処理が変わってはいけないためクラス属性だが、実際にプログラミングをするときにはインスタンス属性のようにアクセスする。

<a id="13-3-3-1-2"></a>
##### 13.3.3.1.2. インスタンス属性
インスタンス属性はインスタンスごとに値が変わる属性である。インスタンス属性はインスタンスデータ属性を持ち、インスタンスデータ属性の一種にモジュール内関数がある。

| 属性 | 書き方, 例 | 意味 |
| --- | --- | --- |
| データ属性 | `myinstance.name` | インスタンスごとに異なる値 |
| モジュール内関数 | `re.sub(str)` | 厳密にはデータ属性の一種 |

モジュール内関数はメソッドではない。なぜなら、各モジュールは`ModuleType`クラスのインスタンスだが、モジュール内の関数はそのモジュールで定義された処理であって、クラス全体で共通した処理ではないからである。

<a id="13-3-3-1-3"></a>
##### 13.3.3.1.3. クラス属性とインスタンス属性のコードでの説明

```python
class Product:
    tax_rate = 0.1  # クラスデータ属性：消費税率（全商品共通）

    def __init__(self, name, price):
        self.name  = name   # インスタンスデータ属性：商品ごとに異なる
        self.price = price

    def price_with_tax(self):  # メソッド属性
        return self.price * (1 + Product.tax_rate)

apple  = Product("apple",  200)
banana = Product("banana", 150)

print(apple.price_with_tax())   # 220.0
print(banana.price_with_tax())  # 165.0

# 税率が変わったらクラス側だけ変えれば全商品に反映
Product.tax_rate = 0.15
print(apple.price_with_tax())   # 230.0
print(banana.price_with_tax())  # 172.5
```

<a id="13-3-3-1-3-1"></a>
###### 13.3.3.1.3.1. dataclassを使った場合の書き方
dataclassを使うと各フィールドの型を明示することができる。`ClassVar`でクラスデータ属性であることを型として明示する。

```python
from dataclasses import dataclass
from typing import ClassVar

@dataclass
class Product:
    tax_rate: ClassVar[float] = 0.1  # クラスデータ属性：ClassVarで明示

    name: str
    price: float

    def price_with_tax(self) -> float:
        return self.price * (1 + Product.tax_rate)

apple  = Product("apple",  200)
banana = Product("banana", 150)

print(apple.price_with_tax())   # 220.0
print(banana.price_with_tax())  # 165.0

Product.tax_rate = 0.15
print(apple.price_with_tax())   # 230.0
print(banana.price_with_tax())  # 172.5
```

<a id="13-3-3-2"></a>
#### 13.3.3.2. メソッド属性の具体例
メソッド属性は、見かけ上メソッド関数の第1引数が省略されているように見える。<br>
つまり、`A.method_func(b, c)`は`method_func(A, b, c)`だと見なすと分かりやすい。<br>
よく使う`str`型(=`str`クラス)のメソッドをいくつか挙げる。
```python
example = "  hello world  "
print(example.strip())      # "hello world"（前後の空白を削除）
print(example.upper())      # "  HELLO WORLD  "（大文字に変換）
print(example.replace("world", "python"))  # "  hello python  "（置換）
print(example.split())      # ['hello', 'world']（スペース区切りで分割）
```

これらのメソッドは全て`str`型のインスタンスに対してのみ呼び出せ、そのインスタンスに対して機能する。

メソッド関数はメソッドではなく単なる関数として適用することができる。<br>
つまり、`A.B()`は`B(A)`に変換することができる。<br>
しかし、`B(A)`ができても`A.B()`ができない場合はある。例えば、<br>
`int("25")`(`int`型の25が返ってくる)は成立するが`"25".int()`は成立しない。これは`int()`は型変換をするものであり、`str`は`int()`という自身を型変換させるメソッドを持っていないからである。

<a id="13-3-4"></a>
### 13.3.4. インスタンスが`bool`(真偽値)を持つ ― `__bool__()`メソッド
実はインスタンスはbool値を持っていると見なすこともできる。というのも、インスタンスは`__bool__()`メソッドを持っているため、任意のタイミングで`__bool__()`を呼び出して自身に関してのbool値を与えることができるからである。

各型の`__bool__()`メソッドは、「自分自身(インスタンス)が空でないか・ゼロでないか」を判定する処理を持っている。例えば
```python
a = 0
a.__bool__()    # False

a = 1
a.__bool__()    # True

a = ""
a.__bool__()    # False

a = "a"
a.__bool__()    # True

a = []
a.__bool__()    # False

a = [0]
a.__bool__()    # True  ← 中身が0でもリスト自体は空でないのでTrue

a = None
a.__bool__()    # False
```

| 型 | `False`になる場合 | `True`になる場合 |
|---|---|---|
| `int` / `float` | `0`, `0.0` | それ以外の数値 |
| `str` | `""` (空文字列) | それ以外の文字列 |
| `list` / `tuple` / `set` / `dict` | 空(`[]`, `()`, `set()`, `{}`) | 要素が1つ以上ある |
| `NoneType` | 常に`False` | (なし) |

<a id="13-3-5"></a>
### 13.3.5. データ属性が返り値を持てる
メソッド属性`A.B()`は`B()`が関数なため返り値を持つことは理解しやすいが、実は`A.C`のようにデータ値属性も返り値を返すことがある。

例えば、以下のようなDataFrameを考える：
``` python
df_day = pd.DataFrame({
    "date": ["2024-01-15", "2024-02-20", "2024-03-10"],
    "sales": [100, 150, 200]
})
```

行列で表現するなら以下のようになっている

|     | date       | sales |
|-----|------------|-------|
| 0   | 2024-01-15 | 100   |
| 1   | 2024-02-20 | 150   |
| 2   | 2024-03-10 | 200   |

このとき`df_year = df_day["date"].dt.year`という処理は、
``` python
df_year = df_day["date"].dt.year
```
①`df_day`という`DataFrame`が`"date"`という列名の(数)値群（`Series`型）を持っている（`df_day["date"]`）<br>
②その列は全て日付形式であり、`.dt`というaccessor（データ属性）が`DatetimeProperties`というインスタンスを返す<br>
③その`DatetimeProperties`インスタンスの`.year`というプロパティ(データ属性)が、年の部分だけを抽出した新しい`Series`インスタンスを返し、それに`df_year`というラベルをつけている。<br>
という流れになっている。

`=`という代入文の性質上、右辺はなんらかのインスタンスになっているはずなので、`.year`がなんらかのインスタンスを返していることがここから読み取れる。実際に今回は`Series`クラスのインスタンスを返している。

<a id="13-3-5-1"></a>
#### 13.3.5.1. アクセッサ（accessor）とプロパティ（property）
`.dt`と`.year`はaccessorとプロパティと、プログラミング的に異なる性質がある。というのも、`.dt`と`.year`はそれぞれのデータ属性が入っているクラスが異なる。

`.dt`はaccessorで、`DataFrame`クラスに存在するデータ属性であり`DatetimeProperties`のインスタンスを返す。一方、`.year`は`.dt`が返したそのインスタンスのプロパティであり、これが最終的に`Series`インスタンスを返す。

プロパティとaccessorは相補的な関係にあり、以下の関係になっている

- accessor: プロパティ提供するもの。<br>
`.dt` accessor は `DatetimeProperties` というインスタンスを返し、その中に `.year`, `.month`, `.day`, `.hour` といった複数のプロパティが入っている。

- プロパティ: accesorが返すインスタンスに存在するデータ属性であり、実務として意味のある値を返す。

つまり、プロパティは2段階目の属性で、それを呼び出すための鍵がaccessorと見なせる。<br>
また、accessorは「その後の処理を待つ存在」という点でvimのoperatorのようなものと捉えることもできる。

<a id="13-4"></a>
## 13.4. 演算子(operator)について
演算子とは、コードの中でそれを用いたときに特定の処理をする記号のことである。

例えば、`a+b`における`+`は`a`に対して`b`を加えるという処理をさせる記号である。そして、 [[#属性(attribution)という概念]] に書いてある`a.b`というのは「`a`という(クラス)構造体の`b`という属性にアクセスする」という意味を持つ演算子である(属性参照演算子 という名称がある)。

また、`+`や`.`は処理が1つに決まっているが、演算子の中には「この記号はこの処理」というように1対1対応で処理が決まっているような演算子だけでなく、**前後の構文によってどのような処理をするかに決まる演算子もある*。**

なお、`=` は演算子ではない。というのも、`=`はあくまで名付け(参照するオブジェクトを指定)しているのみである。
また、辞書や集合を定義するときの`A= {}`の`{}`も演算子ではない。これはリテラル構文と呼ばれるものである。

演算子の例としては以下がある。

**`[]`については僕の理解が浅いので間違ったことを書いている可能性があります。**

| 演算子  | 処理 |
| --- | --- |
| `.`  | 構造体の属性にアクセスする | 
| `[]` | リストを定義したり辞書構造体やリストの要素を取り出したり、新たに追加する。<br>[i]`A["happy"]`だと`A`という辞書構造体の`happy`というキーのアイテムを取り出す。<br>[ii]`A["happy"]= 2`だと`A`という辞書構造体に`happy`というキーとそれに対応する値`2`を追加する。<br>[iii]`A[n]`だとリストの`n+1`番目の要素にアクセスする。<br>[iv] `A= [a, b, c]`だと`A`をリストオブジェクトだと定義する。 |
| `==` | 後ろのオブジェクトを前のオブジェクトと比較して一致していた場合に`true`を、一致していない場合に`false`を返す。 |

<a id="13-4-1"></a>
### 13.4.1. 演算子と関数
演算子を「特定の処理をする記号」と説明したが、「特定の処理をする」と聞いて思いつくのは関数ではないだろうか。<br>
実際、演算子というものは内部的にはメソッド関数の呼び出しをしているのである。<br>

例えば<br>
`c = a + b`というのは<br>
「`a`という名前が指すインスタンスが数字に関する型であり、そのインスタンスが`__add__()`メソッドを持っているとき、`+`の後ろにあるオブジェクトを引数として_`a.__add__(b)`を行う」という意味なのである。

なお、属性参照演算子の`.`は`__getattribute__()`メソッドであるが、パース(演算子をメソッドに直す作業が含まれる)をしてから再度呼び出しをすることはなく、<br>
`obj.A = obj.__getattribute__("A")= obj.__get_attrbute__(__getattribute__(A""))`のように再帰的に`.`が展開されることはない。

そして、演算子が関数なのだとしたら、関数は返り値を持つのだから演算子も返り値を持つはずである。

例えば、比較演算子`==`は「同じかどうかを確かめる」というのは国語的には正しいが、プログラミングの内部処理を表現していない。<br>

プログラミング的には「`==`の左右が同じインスタンスを指しているかを比べて、同じなら`True`を、異なるなら`Flase`を返す(つまり返り値の型は`bool`)」という役割をしている。

その他の`>`や`<`も`!=`も全て「比べているもの」ではなくて「比べた結果`bool`値、つまり`True`か`False`を返すもの」なのである。

なお、 `not`も演算子であり、機能は後ろにある様々なオブジェクトの持つ`x.__bool__()`を呼び出して、そこで返ってくる`True, False`を反転させるというものである。<br>
実は`not`は対応するメソッド名があるわけではない。インタープリタが直接利用している。

<a id="13-5"></a>
## 13.5. importの仕方
importとは、簡潔に言えば、外部のツールや処理を取り込むことである。

理論上はimportというのが無くてもプログラミングはできるが、実務的には不可欠と言ってよく、責務の分離に強く関わる。

<a id="13-5-1"></a>
### 13.5.1. ライブラリのimport

初心者が目にすることが多いimportはライブラリのimportである。<br>
ライブラリとは、狭義のパッケージあるいはモジュールあるいはバイナリとそのラッパーのことであるが、そこの区別をしなくても済むように(しなくてもいい場合が多いため)ライブラリという用語が使われることが多い。<br>
(このあとパッケージとモジュールの違いについて説明する。)

ライブラリのimportの実務的な意味としては、「標準のそのプログラミング言語にはない機能を提供する」というものである。

例えば、プログラミング言語をデフォルトで高機能にしすぎると使わない機能も読み込んで動作が重くなる。<br>
また、プログラミング言語の開発者ではない人が「こういう機能を作ったから他の人にも使えるようにしたいな」と思うこともある。

こういう場合にプログラミング言語の標準外の機能をライブラリとして公開して、その中で欲しいものを各人が取り込んで使えるようにする処理が`import`である。

下のはPythonのimportの最もシンプルな例である。下の`requests`, `pandas`, `pathlib`の3つのライブラリはそれぞれAPI, データ処理, パスの処理に用いるライブラリである。

```python
import requests
import pandas as pd # pandasパッケージはこのファイルではpdとして表記される
from pathlib import Path
```
`requests`, `pandas`はパッケージである。<br>
一方で`pathlib`ライブラリに関してはパッケージではなくモジュールである。`Pathlib`モジュールのうち`Path`クラスのみを`import`している。

`import`の最も基本的な使い方はライブラリ全体を`import`するか、`from`を使ってライブラリ内の特定のクラスや関数を`import`するやり方である。

<a id="13-5-2"></a>
### 13.5.2. パッケージ(`dir/`)→モジュール(`.py`)→関数, クラス(`function()`, `Class`)

自分で作成したコード（パッケージやモジュール）を`import`する場合、ライブラリのimportよりもやや複雑になる。1つの`.py`の中を考えるだけでなく、`.py`自体や、それらの集合にまで階層を上げて考える必要性が出てくる。

Pythonのコードはパッケージ, モジュール, 関数・クラスという、3段階の階層を持つ。

- パッケージは狭義では`.py`が集合したディレクトリ（`dir/`）である。任意のディレクトリがパッケージとして認識されるためには、そのディレクトリに`__init__.py`というファイルを置く必要がある。<br><br>

広義の(or 別の意味の)パッケージはバイナリとそれを呼び出す`.py`が入った`dir`である。例えば`pandas`は`.py`の集合ではないが、一般的にパッケージと呼ばれる。<br>
C言語で実装してコンパイルしてバイナリにしたものを`.py`が呼び出す形式にしているため高速に動作する。

- モジュールは1つ1つの`.py`ファイルである。

- 関数やクラスはモジュール、すなわち`.py`ファイル内に定義され、`.py`インスタンスに紐づいたインスタンス属性である。厳密には違うが、`.py`をインスタンスとしたときに、モジュール内の関数はメソッドのような使用感になる。

例としては下のようになる。
```
# math モジュールのimport
import math 

# モジュールの関数
math.sqrt(16)      # → 4.0
math.floor(3.7)    # → 3

# クラスのメソッド
greeding = "hello"
greeding.upper()          # → "HELLO"
greeding.replace("h", "H") # → "Hello"
```

<a id="13-5-2-1"></a>
#### 13.5.2.1. 厳密にはモジュール内関数はメソッドではない

上の説明ではモジュール内の関数はメソッドのような使用感だと述べたが、実際使用感が似ているだけでメソッドではない。<br>
[クラス属性とインスタンス属性](#13-3-3-1)で述べているが、さらに詳しく説明する。
モジュール内関数がメソッドではないことは以下の2点から説明できる。

- メソッドはインスタンス属性ではなく、クラス属性である。(インスタンス属性はインスタンスによって値が決まるが、メソッドの動作はインスタンスに依らず、クラス全体で共通である。)

- モジュールというインスタンスは`ModuleType`クラスのインスタンスだが、`ModuleType`クラスにユーザーがモジュール内で定義した関数はメソッドとして登録されていない。


また、モジュール内のクラスを呼び出すこともメソッドとは言わない。

例えば`mymodule.py`があったとき、実際にはモジュール内の`func()`という関数へのアクセスは`mymodule.__dict__["func"]()`が内部的には起きていて、メソッド属性というよりはインスタンスデータ属性である。

<a id="13-5-3"></a>
### 13.5.3. パッケージ内の関数の呼び出し方
`import`するときは、パッケージ, モジュール, メソッドの階層構造に沿ってパッケージからモジュールまで指定する必要がある。

重要なのは、**importはモジュール（`.py`ファイル）の階層まで到達させる**ということである。一度モジュールをimportすれば、そのモジュール内の関数やクラスは`モジュール.関数()`という形でアクセスできる。


以下のようなディレクトリ構造であったとする。
```
main_dir/
├─ main.py
└─ mypackage/                 # パッケージ（ディレクトリ）
   ├─ __init__.py             # パッケージ初期化（公開APIの取りまとめにも使える）
   ├─ A.py           # モジュール（.pyファイル）
   │  ├─ function_A_C()                # 関数
   │  └─ function_A_D()                # 関数
   └─ pakage_sub_dir
      └─ B.py             # モジュール
         ├─ function_B_E()           # 関数
         └─ function_B_F()           # 関数

```

ここで、`main.py`で`function_A_C()`と`function_B_E()`を呼び出す方法を述べる。

とりあえず、重要なのは **`import`では`.py`まで届かせる** ということである。<br>
例えば、上のディレクトリ構造で`main.py`で

```
from mypackage import A
from mypackage.package_sub_dir import B

A.function_A_C()
B.function_B_E()
```

あるいは

```
import mypackage.A
import mypackage.package_sub_dir.B

mypackage.A.function_A_C()
mypackage.package_sub_dir.B.function_B_E()
```

などと書く。

また、`numpy`は`import`のときに`from`もオブジェクトの連続参照もせずに`import numpy`のみであらゆるモジュールを用いることができる。<br>
これは`__init__.py`内でも`import`を行うことで実現できる。例えば

```
---
__init__.py
---

from . import A # .は__init__.pyが含まれるディレクトリ(mypackage/)を指す。
from .package_sub_dir import B
```

とすることで

```
---
main.py
---

import mypackage

mypackage.A.function_A_C()
mypackage.B.function_B_E() # 注意 package_sub_dir は入れない
```

などのようにオブジェクト(属性)の参照を連続で行う形式で呼び出しを行うことができる。

さらに、`__init__.py`に途中の階層から末端の階層の`import`をしておくことでオブジェクトの連続参照における途中の部分を省略して、パッケージから直接関数を呼び出せる。
例えば
```
---
__init__.py
---

from .A import function_A_C, function_A_D
from .package_sub_dir.B import function_B_E, function_B_E
```

のようにすると、

```
---
main.py
---

import mypackage

mypackage.function_A_C()
mypackage.function_B_E()
```

のように書ける。

**注意**
- `from import`の形式で`import`の対象が複数個ある場合、`from`側でオブジェクト参照で深くまで潜ることはよいが、`import`側ではオブジェクト参照はできない。

- from側のオブジェクト参照ではパスの指定のときの`/`は用いない。つまり、ディレクトリ→ディレクトリ, ディレクトリ→ファイル, ファイル→関数 は全て等しく.のみで表す。
