# 5. パソコン基礎知識

<a id="5-1"></a>
## 5.1. OSの話をしよう ― OSは宇宙法則である。
パソコン講座は終わったのでちゃんと勉強っぽい内容に入っていきます。<br>
まずはOSというやつの話からしましょう。

OSというのはOperating Systemの略です。まあ覚えてなくていいです。<br>
そして、おそらく多くの人はWindowsのパソコンを使っているのですがWindowsというのがOSです。MacBookを使っている人はMacOSが入っていて、おそらく入れてる人はいないと思いますがLinuxというOS(+の総称)もあります(Ubuntuとか僕が最近ハマってる好きなNixOSとかはLinuxです)。

そして、Windows, MacOS, LinuxだののOSは一体何なのかというと、**OSは宇宙法則**だと思ってほしいです。<br>
「OSというのが世界の在り方、例えば世界の構成だったり物理法則を規定しており、OSによって構成や物理法則が異なる」みたいな感じだと思ってください。

とはいえ、私たち一般人は「このOSが使いやすい」とかそういうのはあっても「このOSはどんな法則が規定されてるんだ」的な内部のことはあまり考えなくていいです(構成は多少考えた方がいいんですが)。それを勉強するのは完全にコンピューターサイエンス(CS)系の勉強になります。

## 5.2. ディレクトリの話をしよう ― <code>C:</code>が宇宙, <code>~</code>が星
OSの話が終わったのでディレクトリの話に入ります。

<a id="5-2-1"></a>
### 5.2.1. dir構造について ― ディレクトリ≒ フォルダ
では、プログラミングをやる上で絶対に外すことができない概念、ディレクトリについて説明していきます。

まず、**ディレクトリというのはいわゆるフォルダのことです。** 今後は省略してdirという表記をしていきます。

そして上で「OSごとに世界の構成が異なる」と書いたのですが、その世界の構成というのがディレクトリ構造というものだと思ってください。  
**ディレクトリ構造というのは、「このディレクトリ(フォルダ)の下にはフォルダ Aとフォルダ Bとフォルダ Cがあって、フォルダ Aにはこのファイルが入ってる」みたいなことです。**

具体例として、Windowsのホームディレクトリ直下のdir構造で説明します。

**Windowsという宇宙法則で作られている宇宙があるとしたときにホームディレクトリ(home dir)が星の最高司令部みたいなものです。**<br>
宇宙の最高司令部がC:なので、「宇宙の最高司令部ほどは権限が強くないけど、でもかなり色々できる」みたいに思ってください。

Windowsにおけるhome dirは以下のようになっています。
ぜひエクスプローラーというソフトで「PC → 謎の文字列+(C:) → Users → 自分のユーザー名(publicとかじゃないやつ)」とクリックしていけば下のと同じのが出てくるはずなので、クリックしながら「ほんまや」って理解していきましょう。

```text
/C:/Users/<ユーザー名>/ (= ~)
├─ Desktop
├─ Documents
├─ Downloads
├─ Music
├─ Pictures
├─ Videos
├─ AppData
│  ├─ Local
│  ├─ LocalLow
│  └─ Roaming
├─ OneDrive
├─ .local (最初の状態では存在しない)
│  └─ bin  
└─ (そのほかユーザーが作成したフォルダ)
```
<br>

そして、プログラミングをやっているとよく出ている`~`はhome dirのことです。  このあと`~`については説明します。

また、Windowsの宇宙が始まるところ, かつ最高司令部を`C:`, Cドライブと言います。<br>
宇宙の最高司令部である`C:`から見たdir構造は下のようになっています。

```text
/C:/
├─ Program Files
├─ Program Files (x86)
├─ Users
│  └─ <ユーザー名>  (= home dir)
│      ├─ Desktop
│      ├─ Documents
│      ├─ Downloads
│      ├─ Music
│      ├─ Pictures
│      ├─ Videos
│      ├─ AppData 
│      └─ .local/ (最初の状態では存在しない)
│         └─ bin/
├─ Windows
├─ ProgramData
├─ PerfLogs
└─ (環境によって追加のフォルダ)
```

<a id="5-2-2"></a>
### 5.2.2. dirの表記 ― パス, パソコン内の場所を記述できるようにしよう。
察しの良い人は気づいたかもしれませんが、dir構造は住所のようになっています。そして、住所であるならばそれぞれのdirやファイルの場所を文字列で表現することができるはずです。その文字列のことをパスといいます。<br>
(厳密には、パスは文字列ではなく住所そのものです)

今から、宇宙の最高司令部の`C:`から見たdir構造や、星の最高司令部であるhome dirのdir構造を使って、場所(path)の書き方を整理します。  

<a id="5-2-2-1"></a>
#### 5.2.2.1. 絶対パス ― 神の世界から見た住所
パスの書き方には「絶対パス」と「相対パス」の2つの形式が存在します。実務上は相対パスの方が使いやすいのですが、dir構造を覚えるためにも絶対パスから説明します。

**絶対パスというのは「神の世界から見た」パスです。** 神の世界というのは`C:`のさらに上の階層です。全ての宇宙を見渡せる階層みたいな感じに思ってください。<br>
(詳しくは下の [補足: なぜ最初に <code>/</code> を付けるのか。](#why-leading-slash)を参照してください。)

**例として`C:`から見たdir構造で説明します。**
```
C:/
├─ Windows
├─ ProgramData
└─ [その他色々]
```
に対して、**`ProgramData`というdir(フォルダ)は`C:/ProgramData/`とか`/c/ProgramData/`とかって書きます。**

Windowsの宇宙法則では`C:\ProgramData\`のように`\`(バックスラッシュ)で書くように内部でなっていますが、MacやLinuxをはじめとして、**基本的にプログラミングの世界では`/c/ProgramData/`のような表記をすることが多いです。** Windowsでもターミナルで`bash`や`zsh`というものを使う時にはこの書き方なのでこれで説明します。<br>
(詳しくは[ターミナルとシェルの違い ― コマンドプロンプトもPowerShellもはターミナル言語](#8-0-4-1) を見て下さい)

なお、**`/c/ProgramData/`と`/c/ProgramData`のように、最後に`/`をつけてもつけなくても大丈夫です。** 僕はつけたりつけなかったりですが、最初の方は「これはdirなんだ」ということを明示するために`/`をつけておくと良いような気はします。

<details id="why-leading-slash">
  <summary>補足: なぜ最初に <code>/</code> を付けるのか。</summary>
    まず、OSは宇宙という話をしましたが、パソコンは様々な宇宙を内包させることができます。MacOSはApple製品にしか載っていないだけで、パソコンに複数のOSを導入することは可能なんです。よくあるのはWindowsパソコンにLinuxを載せるやつです(WSLというWindowsの宇宙に仮想のLinuxの宇宙を創るやつとは違います)。そして、<code>/c</code>というのは、パソコンと言う神の世界から<code>(神の世界)/c</code>という特定の宇宙の世界(この例ではWindowsの宇宙)にアクセスしていることを意味していると捉えることができます。
</details>

**次にhome dir配下のパスの話をします。**

home dirは`/c/Users/[ユーザー名]/`というパスになっています。そのため、ダウンロードディレクトリ(フォルダ)は`/c/Users/[ユーザー名]/Downloads`となります。

ダウンロードディレクトリに`tuat_daisuki.md`というファイルがあるならば、このファイルのパスは`/c/Users/[ユーザー名]/Downloads/tuat_daisuki.md` になりますよね。

**ちなみに、エクスプローラー(黄色と水色のアプリ)では「ダウンロード」のように日本語で書かれていますが、内部的には英語で登録されているので英語で書くようにしましょう。** また、Downloadsでもdownloadsでも大丈夫です。他のデフォルトのdirも1文字目が小文字でも大丈夫です。

```
/C:/
├─ Program Files
├─ Program Files (x86)
├─ Users
│  └─ <ユーザー名>  (= home dir)
│     ├─ Desktop
│     ├─ Documents
│     ├─ Downloads
│     ├─ Music
│     ├─ Pictures
│     ├─ Videos
│     ├─ AppData
│     └─ .local (最初の状態では存在しない)
│        └─ bin
├─ Windows
├─ ProgramData
├─ PerfLogs
└─ (環境によって追加のフォルダ)
```

#### 5.2.2.2. 展開パス ― `~`が便利！！
そして、home dirには特殊な性質があります。それが、`~`というホームチルダと僕が呼称しているやつです。<br>
というのも、プログラミングにおいては 必要なファイルなどは全てhome dir配下、つまり星の中で管理するのが美しいとされています。宇宙に置くのはあまり推奨されません。

そして、星の中で管理する運用を心がけると、ユーザーはhome dirより下(配下といいます)での操作が多くなりますよね。それを毎回真面目に書くと`/c/Users/[ユーザー名]/Documents/`などのようになってしまいとっても煩わしいですよね。

また、他のパソコンを触るときにいちいちそのパソコンのユーザー名に合わせて打つのは面倒くさいですし再現性が低いですよね。

**そのため、頭の良い人が`/c/Users/[ユーザー名]/`の部分を`~`で代替するように設計してくれています。**<br>
つまり、`~/Downloads/test.txt` は `C:/Users/<ユーザー名>/Downloads/test.txt` と同じです。

<details id="home-dir-handling"><summary>パスを扱うプログラムの<code>~</code>の扱い</summary>
今までした<code>~</code>の話はターミナル(シェル)の話であって、パスを使ってファイルを動かしたりコピーしたりするプログラムを自分で作った場合には自動で`~`が展開されるわけではないので、「`~`があったらホームチルダとして展開しようね」みたいな設定が必要になります。
</details>
<br>

##### 5.2.2.2.1. デフォルトの`cd`, `z`の挙動 ― home dirに移動する。
ちなみに、home dir配下(=より下の階層)がパソコンの世界でよく用いることを表す事実として、**dirを移動するときに用いる標準コマンドの`cd`や、僕が大好きな外部導入コマンドの`z`(`zoxide`)が、引数を取らないときにhome dirに移動する** というのがあります。

この後コマンドについての解説は行うのですが、`cd`について簡単に動作を説明します。`cd`はchange dirの略(由来)です。<br>
(コマンドについて勉強したい方は [移動と確認 ― cd, ls](#9-3-1) を見て下さい。)

例えば下のような ファイルを動かすコマンドがあるとしましょう。
```
hirose-suguur@everyday-happy ~  # ~とあるので今はhome dirにいます
$ mv Documents/農工大/1年前期/情報処理学/test.txt Documents/農工大/1年前期/細胞・遺伝学/
```
(`mv`はmoveの略で、dirやファイルを動かすコマンドです。`情報処理学/`にあるファイルを`細胞・遺伝学/`に動かしています。)  

上のコマンドは2回も`Documents/農工大/1年前期`と打っていて、無駄が多い気がしますよね。

一方で`cd`を使うと
```
hirose-suguur@everyday-happy ~
$ cd Documents/農工大/1年前期/

hirose-suguru@everyday-happy ~/Documents/農工大/1年前期/ # ←移動しました
$ mv 情報処理学/test.txt 細胞・遺伝学/
```
とするだけでよくなります。簡単に言うと、ある程度まで同じdirのファイルやdirの操作をするときに、先に`cd`を使うと入力が楽になります。

そして、その`cd`は
```
hirose-suguur@everyday-happy ~
$ cd Documents/農工大/1年前期/

hirose-suguru@everyday-happy ~/Documents/農工大/1年前期 
$ cd

hirose-suguru@everyday-happy ~
$
```
のように2つ目のコマンドのようにどこかのdirに移動した後で`cd`の後ろに何もつけずに`cd`をするとhome dirに移動します。これはhome dirより上の階層(宇宙の方)にいても同じです。

<a id="5-2-2-3"></a>
#### 5.2.2.3. 相対パスについて ― 実務的にはこっちを使うことが多い。
先ほどまでは 宇宙を超えた神の世界を基準にパスを書く絶対パスを説明しました。<br>
**次は今いる場所を基準にした相対パスの説明をします。** 先ほどから述べている通り、プログラミング的なことをするときには相対パスの方が遥かによく用いられます。

まず、実は既に相対パスを用いています。それが下の例です。
```
hirose-suguur@everyday-happy ~
$ cd Documents/農工大/1年前期/

hirose-suguru@everyday-happy ~/Documents/農工大/1年前期
$ mv 情報処理学/test.txt 細胞・遺伝学/
```
<br>

この2つ目のコマンドは`~/Documents/農工大/1年前期`を基準としたときに、すぐ下に(直下と言います)`情報処理学/`や`細胞・遺伝学/`があるので直接アクセスしています。

これは`~/Documents/農工大/1年前期/`を基準とした相対パスで指定しているわけです。

また、`cd`が1階層浅いならば
```
hirose-suguur@everyday-happy ~
$ cd Documents/農工大/

hirose-suguru@everyday-happy ~/Documents/農工大
$ mv 1年前期/情報処理学/test.txt 1年前期/細胞・遺伝学/test.txt
```
のようになります。

ここまでは直感に即しているので分かりやすいと思いますが、2つだけ相対パスの独特の記法があります。それが以下の2つです。
- **`./` : 今いるdir (current dir, カレントdir)。**
- **`../` : 1つ上のdir**

**(どちらも`/`が無くてもいいですが`..`を連続して使うときは`../..`のように間に`/`を入れないといけません)**

これを駆使することで全てのdirを相対パスで表せるようになります。

例えば、今`~/Documents/農工大/`にいるとして、  
`~/Pictures/スクリーンショット/スクリーンショット(1)`は  
`..(これでDocuments)/..(これでhome dir)/Pictures/スクリーンショット/スクリーンショット(1)`のように表せます。

また、今いるdir (current dirと言います)は何に使うのかと思うかもしれませんが、コマンドで使うことが多いです。
例えば、`git`というシステムのコマンドを用いる場合にステージングという操作があるのですが、そのときに
```
git add .
```
という操作をよくします。
この`.`(`./`でもいいです)が「current dirより下の全てのファイルを対象として」を意味するわけです。
